Отчет «Обеспеченность плановой потребности в ТМЦ»

Отчет предназначен для вывода информации о необходимых материальных затратах на выполнение плановых ремонтов за указанный период и текущих остатков на складе.
Открывается по команде Обеспеченность плановой потребности в ТМЦ: в списке Отчеты по МТО подсистемы МТО. 

Параметры отчета 

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:
  • Период отчета - отбор ремонтов по плановым датам;
  • Учитывать план графики ППР - настройка, позволяющая учитывать ремонты из документов "План-график ППР";
  • Учитывать регламентные мероприятия - настройка, позволяющая учитывать ремонты из документов  "График регламентных мероприятий";
  • Учитывать Сметы ремонта (Заявки на ремонт) по дефектам - настройка, позволяющая учитывать ремонты из документов "Выявленные дефекты";
  • Учитывать Сметы ремонта (Заявки на ремонт) по внешним основаниям - настройка, позволяющая учитывать ремонты из документов "Внешнее основание для работ";
  • Исключать аварийный запас - настройка, позволяющая учитывать аварийный запас при расчете необеспеченной потребности.

Варианты отчета

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Существует один предопределенный вариант отчета - Обеспеченность плановой потребности в ТМЦ. Данные сгруппированы по складу и номенклатуре. Фиксация плановых ремонтов осуществляется документами "Смета ремонта (Заявка на ремонт)", "План-график ППР", "График регламентных мероприятий".

Основные поля отчета:
  • Склад - наименование склада;
  • Номенклатура - наименование номенклатуры, которая используется для выполнения ремонтной работы (поле "Номенклатура" из документа "Смета ремонта (Заявка на ремонт)" на закладке "Материальные затраты" и Запчасти" для документов "Внешнее основание для работ", "Выявленные дефекты"; поле "Номенклатура" согласно полю "Технологическая карта" из справочника "Объекты ремонта" на вкладке "Нормативы планирования - Нормативные ремонты/Запчасти" для документов "План-график ППР", "График регламентных мероприятий");
  • Характеристика - наименование характеристики номенклатуры (выводится, если установлена настройка системы в разделе Настройка параметров системы - Интерфейс и ввод документов - МТО: Использовать характеристики номенклатуры);
  • Объект ремонта - наименование объекта ремонта;
  • Документ-источник - ссылка на документ-источник;
  • Единица измерения - наименование единицы измерения номенклатуры;
  • Вид ремонта - вид ремонта (поле "Вид ремонта" из документа "План-график ППР"; поле "Вид мероприятия" из документа "График регламентных мероприятий"; поля "Вид ремонтных работ" из документа "Смета ремонта (Заявка на ремонт)" для документов "Выявленные дефекты" и "Внешние основания для работ");
  • Количество ремонтов - количество различных ремонтов в документе плановых ремонтов;
  • Потребность - количество номенклатуры, требуемой для выполнения ремонта;
  • На складе - количество остатков номенклатуры на складе;
  • Запланировано поступление - количество номенклатуры, запланированное к поступлению (согласно документу "Заказ поставщику");
  • Аварийный запас - количество минимального остатка номенклатуры для данного склада (согласно документу "Установка минимального остатка номенклатуры");
  • Необеспеченная потребность - количество номенклатуры, которое необходимо обеспечить. Рассчитывается по формуле: Потребность - На складе - Запланированное поступление. С установленным флагом "Исключать аварийный запас" рассчитывается по формуле: Потребность - (На складе - Аварийный запас) - Запланированное поступление.
  • Стоимость - суммарная стоимость необеспеченной потребности; 
  • Валюта - валюта, в которой рассчитывается стоимость необеспеченной потребности. Берется из настройки "Тип цены для расчета плановой себестоимости".
Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
