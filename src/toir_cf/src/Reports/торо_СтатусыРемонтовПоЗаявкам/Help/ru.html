Отчет «Статусы ремонтов по заявкам»

Отчет предназначен для получения информации о статусах ремонтов из заявок за период. Доступен при включенном флаге в разделе Настройка параметров системы - Интерфейс и ввод документов - Планирование ТОиР: Использовать сметы ремонта.
Открывается по команде Статусы ремонтов по заявкам:

  • в списке Отчеты по планированию ТОиР и Отчеты по нарядам и работам подсистем Планирование ТОиР и Наряды и работы;
  • из документа "Смета ремонта (Заявка на ремонт)" по кнопке "Перейти".

Параметры отчета

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. Для фильтра по выводимым данным в отчете можно отредактировать параметры отчета:

  • Период отчета - отбор статусов по дате их изменения в документах;
  • Структура иерархии - отбор, позволяющий вывести полный путь к объекту ремонта в зависимости от выбранной иерархии (поле "Объект иерархии" в вариантах отчетов "Статусы ремонтов по заявкам (с иерархией)" и "Статусы ремонтов по заявкам (по бригадам, с иерархией)"). По умолчанию заполняется из пользовательских настроек (настройка "Основная структура иерархии");
  • Показывать в работе - настройка, позволяющая учитывать ремонты со статусом "В работе"
  • Показывать завершенные - настройка, позволяющая учитывать ремонты со статусом "Выполнен";
  • Показывать зарегистрированные - настройка, позволяющая учитывать ремонты со статусом "Зарегистрирован"
  • Показывать отклоненные - настройка, позволяющая учитывать ремонты со статусами "Отказ подразделения", "Отказ исполнителя" и "Отклонен";
  • Заявка на ремонт - настройка, позволяющая выводить статусы ремонтов по конкретному документу "Смета ремонта (Заявка на ремонт)".

Варианты отчета

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Доступны четыре предопределенных варианта отчета:

  1. Статусы ремонтов по заявкам - открывается по умолчанию. Данные по статусам ремонтов сгруппированы по заявкам, видам ремонта и датам ремонта.
    В этом варианте отчета по умолчанию выводится информация по статусам ремонтов из заявок. Изменение статусов ремонтов выполняется документами "Смета ремонта (Заявка на ремонт)" (при регистрации и по кнопкам на форме документа), "Наряд на выполнение ремонтных работ", "Акт о выполнении этапа работ" и "Закрытие заявок и ремонтов" (формируется автоматически из обработки "Закрытие заявок и ремонтов").
    • Заявка на ремонт - ссылка на документ "Смета ремонта (Заявка на ремонт)";
    • Вид ремонта - вид ремонта (поле "Вид ремонтных работ" из документа "Смета ремонта (Заявка на ремонт)");
    • Объект ремонта - наименование объекта ремонта;
    • Дата начала - планируемая дата начала ремонтных работ (поле "Дата начала" из документа "Смета ремонта (Заявка на ремонт)");
    • Дата окончания - планируемая дата окончания ремонтных работ (поле "Дата окончания" из документа "Смета ремонта (Заявка на ремонт)");
    • Дата изменения статуса - дата и время изменения статуса ремонта;
    • Пользователь - ответственный пользователь, который изменил статус ремонта (поле "Ответственный" из документов "Смета ремонта (Заявка на ремонт)", "Наряд на выполнение ремонтных работ", "Акт о выполнении этапа работ" и "Закрытие заявок и ремонтов");
    • Статус - установленный статус ремонта (поле "Статус" из документа "Смета ремонта (Заявка на ремонт)"). У ремонта может быть 7 статусов:
      • Зарегистрирован - начальный статус ремонта, который устанавливается после проведения документа "Смета ремонта (Заявка на ремонт)";
      • В работе - статус ремонта, который устанавливается после проведения документов "Наряд на выполнение ремонтных работ" и "Акт о выполнении этапа работ" (без завершающего флага);
      • Отклонен - статус ремонта, который устанавливается при отклонении или переносе ремонта/заявки через обработку "Закрытие заявок и ремонтов";
      • Отказ подразделения - статус ремонта, который устанавливается вручную в документе "Смета ремонта (Заявка на ремонт)" (по кнопке "Отказ подразделения" на закладке "Исполнители". Кнопка доступна, если исполнитель ремонта - подразделение и не выбрана бригада);
      • Отказ исполнителя - статус ремонта, устанавливается вручную в документе "Смета ремонта (Заявка на ремонт)" (по кнопке "Отказ исполнителя" на закладке "Исполнители". Кнопка доступна, если исполнитель ремонта - подразделение и выбрана бригада, которая будет выполнять ремонт);
      • Возобновлено после отказа - статус ремонта, который устанавливается вручную в документе "Смета ремонта (Заявка на ремонт)" (по кнопке "Вернуть в работу" на закладке "Исполнители". Кнопка доступна, если статус ремонта "Отказ подразделения" или "Отказ исполнителя");
      • Выполнен - статус ремонта, который устанавливается после завершения ремонта (флаг "Завершить ремонтные работы" в документе "Акт о выполнении этапа работ").
    • Текущий статус - актуальный на текущий момент статус ремонта (поле "Статус ремонта" из документа "Смета ремонта (Заявка на ремонт)");
    • Комментарий к статусу - текстовое описание статуса. Доступно только при установлении статусов "Отказ подразделения", "Отказ исполнителя" и "Возобновлено после отказа" и является для них обязательным для заполнения (поле "Комментарий к статусу" из документа "Смета ремонта (Заявка на ремонт)").
  2. Статусы ремонтов по заявкам (с иерархией) - вариант отчета, в котором данные по статусам ремонтов сгруппированы по объекту иерархии, объекту ремонта, заявкам, видам и датам ремонта.
    В этом варианте отчета по умолчанию выводится информация (как и в варианте отчета "Статусы ремонтов по заявкам") по статусам ремонтов из заявок. Дополнительно выводится полный путь к объекту ремонта:
    • Объект иерархии - наименования всех родителей объекта ремонта (объекты ремонта или группы объектов ремонта).
  3. Статусы ремонтов по заявкам (по бригадам) - вариант отчета, в котором данные сгруппированы по заявкам, видам ремонта и датам ремонта.
    В этом варианте отчета выводится информация (как и в варианте отчета "Статусы ремонтов по заявкам") по статусам ремонтов из заявок, но для которых указан исполнитель ремонта (подразделение):
    • Подразделение - наименование подразделения-исполнителя ремонтных работ (поле "Исполнитель" на закладке "Исполнители" документа "Смета ремонта (Заявка на ремонт");
    • Бригада - наименование ремонтной бригады, которая является исполнителем ремонтных работ (поле "Договор/бригада" на закладке "Исполнители" документа "Смета ремонта (Заявка на ремонт").
  4. Статусы ремонтов по заявкам (по бригадам, с иерархией) - вариант отчета, в котором данные сгруппированы по объекту иерархии, объекту ремонта, заявкам, видам и датам ремонта.
    В этом варианте отчета выводится информация (как и в варианте отчета "Статусы ремонтов по заявкам") по статусам ремонтов из заявок, но для которых указан исполнитель ремонта (подразделение). Дополнительно выводится полный путь к объекту ремонта:
    • Объект иерархии - наименования всех родителей объекта ремонта (объекты ремонта или группы объектов ремонта);
    • Подразделение - наименование подразделения-исполнителя ремонтных работ (поле "Исполнитель" на закладке "Исполнители" документа "Смета ремонта (Заявка на ремонт");
    • Бригада - наименование ремонтной бригады, которая является исполнителем ремонтных работ (поле "Договор/бригада" на закладке "Исполнители" документа "Смета ремонта (Заявка на ремонт").
Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
