Отчет «Выполнение плановых и внеплановых ремонтов»

Отчет предназначен для получения информации о текущих статусах фактического выполнения плановых и внеплановых ремонтов. В отчете выводится информация о плановых датах начала и окончания периода ремонтных работ и о фактических датах периода ремонтных работ.
Открывается по команде Выполнение плановых и внеплановых ремонтов: 
  • в списке Отчеты по планированию ТОиР подсистемы Планирование ТОиР;
  • в списке Отчеты по нарядам и работам подсистемы Наряды и работы.
Параметры отчета
Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:
  • Период отчета - отбор плановых/внеплановых ремонтов по плановым датам;
  • Учитывать план графики ППР - настройка, позволяющая учитывать ремонты из документов "План-график ППР";
  • Учитывать ремонты по дефектам - настройка, позволяющая учитывать ремонты из документов "Выявленные дефекты";
  • Учитывать ремонты по внешним основаниям - настройка, позволяющая учитывать ремонты из документов "Внешнее основания для работ";
  • Учитывать регламентные мероприятия - настройка, позволяющая учитывать ремонты из документов  "График регламентных мероприятий".
Варианты отчета
С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Доступно три предопределенных варианта отчета:
  1. Плановые и внеплановые ремонты: таблица - открывается по умолчанию. Данные о ремонтах сгруппированы по объектам ремонта.
    В этом варианте отчета по умолчанию в строках выводится информация о плановых и внеплановых ремонтах. Фиксация плановых ремонтов осуществляется документами "План-график ППР" и "График регламентных мероприятий", а внеплановых - документами "Выявленные дефекты", "Внешнее основание для работ":
    • Объект ремонта - наименование объекта ремонта;
    • Документ-инициатор - ссылка на документы "Выявленные дефекты", "Внешнее основание для работ", "План-график ППР", "График регламентных мероприятий";
    • Дата обнаружения - дата и время обнаружения дефектов оборудования для документа "Выявленные дефекты" (поле "Дата обнаружения"). Дата и время документа "Внешнее основание для работ" (поле "Внешнее основание от"). Дата и время регистрации документа в системе для документов "План-график ППР" и "График регламентного мероприятия" (поле "от");
    • Ремонт - вид ремонта (поле "Вид ремонта" из документа "План-график ППР"; поле "Вид мероприятия" из документа "График регламентных мероприятий"; поля "Вид ремонта при вводе на основании выявленных дефектов" и "Вид ремонта при вводе на основании внешних оснований" из настроек параметров подсистемы ТОИР);
    • Плановая дата начала - планируемая дата начала ремонтных работ (поле "Дата начала" из документа "Смета ремонта (Заявка на ремонт)" для документов "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР"; поле "Период" из документов "План-график ППР", "График регламентных мероприятий" для "План-график ППР" и "График регламентных мероприятий");
    • Плановая дата окончания - планируемая дата окончания ремонтных работ (поле "Дата окончания" из документа "Смета ремонта (Заявка на ремонт") для документов "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР"; поле "Период" из документов "План-график ППР", "График регламентных мероприятий" для "План-график ППР" и "График регламентных мероприятий");
    • Фактическая дата начала - фактическая дата начала ремонтных работ (поле "Дата начала" из документов "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия");
    • Фактическая дата окончания - фактическая дата окончания ремонтных работ (поле "Дата окончания" из документов "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия" (с завершающими флагами));
    • Статус выполнения - присваиваемый статус ремонта. Ремонту может быть присвоено три статуса:
      • Работы не начаты - начальный статус для всех ремонтов;
      • Работы начаты - статус ремонта, который устанавливается после проведения документов "Наряд на выполнение ремонтных работ","Наряд на выполнение регламентного мероприятия" и "Акт о выполнении этапа работ", "Акт о выполнении регламентного мероприятия" (без завершающих флагов);
      • Работы завершены - статус ремонта, который устанавливается после проведения документов "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия" (с завершающими флагами).
    • Описание - текстовое описание ремонта (поле "Описание" из документа "Внешнее основание для работ"; поле "Комментарий к направлению на устранение" из документа "Выявленные дефекты");
    • Документ акт - ссылка на документы "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия";
    • Количество - количество ремонтов, выполняемых в рамках документа-инициатора в зависимости от выбранного периода;
    • Длительность ремонта - фактическая длительность выполнения ремонтных работ (рассчитывается как разница между Фактической датой окончания и начала работ);
    • Длительность ремонта по плану - плановая длительность выполнения ремонтных работ (рассчитывается как разница между Плановой датой окончания и начала работ);
    • Итого - итоговая сумма количества ремонтов*, длительности ремонтов и длительности ремонтов по плану за выбранный период.
2.  Внеплановые ремонты: таблица - вариант отчета, в строках которого выводится информация о внеплановых ремонтах. Данные о ремонтах сгруппированы по объекту ремонта, ремонту, плановой дате начале. Фиксация ремонтов осуществляется документами "Выявленные дефекты", "Внешнее основание для работ":
    • Длительность устранения дефекта - длительность устранения дефекта (рассчитывается как разница между Фактической датой окончания и датой обнаружения);
    • Время реакции - время реакции (рассчитывается как разница между Фактической датой начала и датой обнаружения).
3.  Плановые ремонты: таблица - вариант отчета, в строках которого выводится информация о плановых ремонтах. Данные о ремонтах сгруппированы по объекту ремонта, ремонту, плановой дате начале. Фиксация ремонтов осуществляется документами "План-график ППР" и "График регламентных мероприятий".
      Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.

Примечания*:
Формула количества ремонтов считает по количеству уникальных id ремонтов. В связи с этим ремонты по документу "График регламентных мероприятий" в итоговой колонке считаются единожды.
