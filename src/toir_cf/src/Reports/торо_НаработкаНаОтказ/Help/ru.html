Отчет «Наработка на отказ»

Отчет предназначен для получения информации о наработке и времени работы до отказа. 
Открывается по команде Наработка на отказ в панели отчетов Отчеты по учету показателей эксплуатации подсистемы Учет показателей эксплуатации. В отчете выбираются только документы по выявленным дефектам (отказам).

Основные поля отчета:

  • Показатель - показатель наработки объекта ремонта, по которому рассчитывается фактическая наработка оборудования.
  • Объект ремонта - объект ремонта, в разрезе которого выводится информация по отказам.
  • Вид ремонта - вид ремонта предыдущего и текущего дефектов. Если по дефекту не введено никаких документов, то значение будет не заполнено.
  • Даты обнаружения и устранения - даты, на основании которых рассчитываются значения (даты обнаружения).
  • Наработка - фактическая наработка оборудования с даты обнаружения предыдущего дефекта по дату обнаружения текущего. Если период фактической наработки не совпадает с периодом дефекта, то в качестве периода берется их пересечение, вычисляется его доля относительного всего периода наработки, и в качестве значения наработки берется та часть, которая соответствует полученной доле.
  • Время работы - время работы оборудования с даты обнаружения предыдущего дефекта по дату обнаружения текущего. Под временем работы подразумевается нахождение оборудования в состоянии с типом эксплуатации Эксплуатация. На уровне детальных записей выводится конкретное значение. На уровне объектов ремонта выводится среднее время работы объекта ремонта.
  • Количество дефектов - количество дефектов в разрезе различных группировок (в зависимости от варианта отчета). Используется для расчета средних значений наработки и времени работы.
В качестве дат начала и окончания периода, за который рассчитывается наработка и время работы, берутся даты обнаружения предыдущего и текущего дефектов соответственно. В качестве даты начала нельзя брать дату устранения предыдущего дефекта, т.к. после его обнаружения оборудование не всегда останавливается сразу, т.е. может работать еще какое-то время. В данном случае не были бы учтены значения за этот период.

Параметры отчета 

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:

  • Период отчета - отбор дефектов по дате обнаружения (дата должна попадать в отчетный период);
  • Структура иерархии - отбор, позволяющий вывести родителей объектов ремонта в зависимости от выбранной иерархии.

Варианты отчета 

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Существует три предопределенных варианта отчета:

  1. Наработка на отказ. Основной вариант отчета. Отображается в панели отчетов при условии одновременного использования учета по наработке и состояниям оборудования (Настройка параметров системы - Интерфейс и ввод документов - Учет показателей эксплуатации: Учет состояния оборудования и Учет наработки оборудования). Если включена только одна из двух настроек, то в панели отображается только один вариант отчета (наработка или время работы соответственно). Детальные записи отчета сгруппированы по объектам ремонта. В качестве детальных записей выступают дефекты оборудования по каждому показателю наработки. Ремонты отображаются только если за определенный период есть хотя бы одно из двух значений (фактическая наработка и время работы). Поля варианта отчета:
    • Объект ремонта.
    • Вид ремонта.
    • Даты обнаружения и устранения.
    • Наработка - выводится только на уровне детальных записей по каждому ремонту. Если у объекта ремонта несколько параметров наработки, то дефект выводится для каждого параметра. Вычислять значения по объекту ремонта некорректно, т.к. у него могут быть параметры с разными единицами измерения.
    • Время работы - если по объекту ремонта есть строки, в которых не заполнено время работы, то при подсчете количества дефектов они все-равно учитываются. Т.е. вне зависимости от того, было ли оборудование в работе, дефект в любом случае учитывается и уменьшает среднее время работы.
    • Количество дефектов - выводится на уровне объектов ремонта. Показывает общее число дефектов по объектам ремонта за отчетный период.
  2. Наработка на отказ: наработка - отчет предназначен для получения данных о фактической наработке оборудования с даты обнаружения предыдущего дефекта по дату обнаружения текущего. Поля варианта отчета:
    • Показатель.
    • Объект ремонта.
    • Вид ремонта.
    • Даты обнаружения и устранения.
    • Наработка - на уровне детальных записей выводятся конкретные значения наработки в разрезе ремонтов. На уровне объектов ремонта и показателей выводится средняя наработка. На уровне объектов ремонта выводится среднее значение только по тем ремонтам, которые отображаются по объекту ремонта для определенного показателя (для одного и того же объекта ремонта для разных показателей может быть разное количество ремонтов, в случае если у каких-то объектов ремонта параметры наработки были введены в разное время).
    • Количество дефектов - выводится как на уровне объекта ремонта, так и на уровне параметра наработки. На уровне объекта ремонта выводится количество дефектов только по ремонтам, по которым есть данные для конкретного параметра наработки (для разных параметров может быть разное количество дефектов по одному и тому же объекту ремонта). На уровне параметра наработки выводится количество дефектов по всем объектам ремонта, относящихся к нему.
  3. Наработка на отказ: время работы - отчет предназначен для получения данных о времени работы оборудования с даты обнаружения предыдущего дефекта по дату обнаружения текущего. Поля варианта отчета:
    • Объект ремонта.
    • Вид ремонта.
    • Даты обнаружения и устранения.
    • Время работы.
    • Количество дефектов - выводится на уровне объектов ремонта. Показывает общее число дефектов по объектам ремонта за отчетный период.

Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
