Отчет «Журнал проведенных анализов причин дефектов»

Отчет предназначен для получения информации о проведенных анализах причин дефектов за указанный период. Доступен при включенном флаге в разделе Настройка параметров системы – Показатели эксплуатации: Использовать анализ причин дефектов. 

Открывается по команде Журнал проведенных анализов причин дефектов: в списке Отчеты по показателям эксплуатации подсистемы Показатели эксплуатации.

Параметры отчета

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:

  • Период отчета - отбор по дате регистрации анализа причин дефекта.

Варианты отчета

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Существует один предопределенный варианта отчета – Журнал проведенных анализов причин дефектов. Данные о проведенных анализах причин дефектов сгруппированы по организациям, подразделениям и ответственным. Фиксация анализов причин дефектов выполняется документом "Анализ причин дефекта":

  • Организация - наименование организации, проводящей анализ причины дефекта;
  • Подразделение - наименование подразделения организации, проводящего анализ причины дефекта;
  • Ответственный - ФИО пользователя, который является ответственным за регистрацию анализа причин дефекта;
  • Дата - дата и время регистрации документа в системе;
  • Документ - ссылка на документ "Анализ причин дефекта";
  • Объект ремонта - наименование объекта ремонта;
  • Дата события - дата и время обнаружения выявленного дефекта объекта ремонта;
  • Выявленный дефект - ссылка на документ "Выявленные дефекты";
  • Комиссия - комиссия, ответственная за проведение анализа причин дефекта;
  • Статус - установленный статус анализа (поле "Статус документа" из документа "Анализ причин дефекта"). Может принимать 4 статуса:
    • Зарегистрирован - начальный статус анализа, который устанавливает после проведения документа "Анализ причин дефекта";
    • В работе - статус анализа, который устанавливается пользователем вручную или автоматически (при заполнении табличной части "Дерево анализа");
    • Анализ выполнен - статус анализа, который устанавливается после заполнения реквизитов документа, а также коренных и непосредственной причин. Документ становится недоступен для редактирования, за исключением таблицы "Корректирующие мероприятия";
    • Закрыт - статус анализа, который устанавливается после того, как все ремонты по корректирующим мероприятиям выполнены (или отменены) и завершены все "Задания" бизнес-процессов;
  • Дата завершения - дата завершения проведения анализа причин дефекта;
  • Количество - итоговое количество выведенных документов в рамках группировки.

Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
