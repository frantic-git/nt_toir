Отчет «Анализ частоты причин дефектов»

Отчет предназначен для анализа частоты возникновения причин дефектов за указанный период. Доступен при включенном флаге в разделе Настройка параметров системы – Показатели эксплуатации: Использовать анализ причин дефектов.
Открывается по команде Анализ частоты причин дефектов: в списке Отчеты по показателям эксплуатации подсистемы Показатели эксплуатации.

Параметры отчета

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:

  • Период отчета - отбор по дате регистрации анализа причин дефекта;
  • Периодичность - отбор, позволяющий отобразить период отчета в разрезе дня/недели/месяца/квартала/года.

Варианты отчета

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Доступны три предопределенных варианта отчета:

  1. Анализ частоты причин дефектов: таблица - открывается по умолчанию. Данные о дефектах сгруппированы по коренным и непосредственным причинам дефекта, а также объектам ремонта. В этом варианте отчета по умолчанию выводятся выявленные дефекты, по которым в анализе причин дефекта указана корневая причина. Регистрация выявленных дефектов и анализов причин дефектов выполняется документами "Выявленные дефекты" и "Анализ причин дефекта":
    • Коренная причина дефекта - наименование коренной причины дефекта;
    • Непосредственная причина дефекта - наименование непосредственной причины дефекта;
    • Объект ремонта - наименование объекта ремонта;
    • Выявленный дефект - ссылка на документ "Выявленные дефекты";
    • Дата события - дата и время обнаружения дефекта оборудования (поле "Дата обнаружения" из документа "Выявленные дефекты");
    • Отказавший элемент - наименование ремонтной единицы, для которой регистрируется неисправность (поле "Отказавший элемент" из документа "Выявленные дефекты");
    • Описание дефекта - краткое описание дефекта (поле "Описание дефекта" из документа "Выявленные дефекты");
    • Критичность дефекта - наименование критичности дефекта (поле "Критичность дефекта" из документа "Выявленные дефекты");
    • Количество - количество дефектов, сгруппированное по коренной причине возникновения дефекта оборудования;
    • Общий % - соотношение коренной причины возникновения дефекта, к общему количеству обнаруженных коренных причин за указанный промежуток времени.
  2. Динамика возникновения причин дефектов: диаграмма - отчет предназначен для анализа динамики возникновения причин дефектов в количественном выражении графическом виде за указанный промежуток времени.
  3. Процентное соотношение причин дефектов: диаграмма - отчет предназначен для анализа динамики возникновения причин дефектов в процентом соотношении в графическом виде за указанный промежуток времени.
Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
