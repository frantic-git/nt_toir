Отчет «Отчет по операциям»

Отчет предназначен для получения информации о выполнении ремонтных работ в процентном соотношении в разрезе плана и факта. Выводится информация о выполнении технологических операций. Сумма выполнение всех операций составляет - 100%.  Чтобы процент выполнения ремонтных работ указывался по операциям, необходимо включить ФО "Использовать усложненную схему закрытия наряда" (в разделе Настройка и администрирование - Настройка параметров системы - Наряды и работы). 
Открывается по команде Отчет по операциям в списке Отчеты по нарядам и работам подсистемы Наряды и работы.
Параметры отчета
Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:
  • Период отчета - отбор ремонтов по плановым датам начала;
  • Структура иерархии - отбор, позволяющий вывести полный путь к объекту ремонта в зависимости от выбранной иерархии. По умолчанию заполняется из пользовательских настроек (настройка "Основная структура иерархии");
  • Учитывать план графики ППР - настройка, позволяющая учитывать ремонты из документов "План-график ППР";
  • Учитывать ремонты по дефектам - настройка, позволяющая учитывать ремонты из документов "Выявленные дефекты";
  • Учитывать ремонты по внешним основаниям - настройка, позволяющая учитывать ремонты из документов "Внешнее основания для работ";
  • Учитывать регламентные мероприятия - настройка, позволяющая учитывать ремонты из документов  "График регламентных мероприятий".
Варианты отчета
С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Доступен один предопределенный вариант - Отчет по операциям. Данные о выполняемых операция сгруппированы по объекту ремонта и виду ремонта. Фиксация плана осуществляется документами -  "Смета ремонта (Заявка на ремонт)", "План-график ППР", "График регламентных мероприятий", факта - документами "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия":

  • Объект иерархии - наименования всех родителей объекта ремонта (объекты ремонта или группы объектов ремонта).
  • Объект ремонта - наименование объекта ремонта; 
  • Вид ремонта - вид ремонта (поле "Вид ремонта" из документа "План-график ППР"; поле "Вид мероприятия" из документа "График регламентных мероприятий"; поля "Вид ремонта при вводе на основании выявленных дефектов" и "Вид ремонта при вводе на основании внешних оснований" из настроек параметров подсистемы ТОИР);
  • Дата начала - дата начала ремонтных работ (поле "Дата начала" из документа "Смета ремонта (Заявка на ремонт)" для документов "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР"; поле "Период" из документов "План-график ППР", "График регламентных мероприятий" для "План-график ППР" и "График регламентных мероприятий");
  • Операция - наименование технологической операции (поле "Ремонтная работа" из дерева ремонтных работ документов "Смета ремонта (Заявка на ремонт)"; поле "Операция/ Ремонт" из дерева ремонтных работ справочника "Технологические карты ремонтов" согласно справочнику "Объекты ремонта" на вкладке "Нормативы планирования - Нормативные ремонты" для документа "План-график ППР"; поле "Операция/ Ремонт" из дерева ремонтных работ справочника "Технологические карты ремонтов" согласно справочнику "Списки объектов регламентных мероприятий" для документа "График регламентных мероприятий");
  • Документ плана -  ссылка на документы, фиксирующие плановые выполнение работ: "Смета ремонта (Заявка на ремонт)" (для документов "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР"), "План-график ППР", "График регламентных мероприятий";
  • Документ-основание - документы, на основании которых создаются документы плана: "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР", "График регламентных мероприятий";
  • Документ факта - ссылка на документы, фиксирующие фактическое выполнение работ: "Акт о выполнении этапа работ", "Акт о выполнении этапа регламентных мероприятий";
  • План, % - фиксируется плановыми документами;
  • Факт, % - фиксируется документами факта;
  • Отклонение, % - рассчитывается как разность между планом и фактом.
Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
