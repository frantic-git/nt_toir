Отчет «История изменения дат ремонта»

Отчет предназначен для вывода информации по изменению плановых дат ремонтов. Данные по ремонтам сгруппированы по объектам ремонта.

Открывается по команде История изменения дат ремонта в списке Отчеты по планированию ТОиР подсистемы Планирование ТОиР.

Параметры отчета 

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:

  • Период отчета - отбор ремонтов по последним плановым датам;
  • Структура иерархии - отбор, позволяющий вывести полный путь к объекту ремонта в зависимости от выбранной иерархии. По умолчанию заполняется из пользовательских настроек (настройка "Основная структура иерархии");
  • Учитывать ремонты по дефектам - настройка, позволяющая учитывать ремонты из документов "Выявленные дефекты";
  • Учитывать план-графики ППР - настройка, позволяющая учитывать ремонты из документов "План-график ППР";
  • Учитывать ремонты по внешним основаниям - настройка, позволяющая учитывать ремонты из документов "Внешнее основание для работ";
  • Учитывать регламентные мероприятия - настройка, позволяющая учитывать затраты из документов  "График регламентных мероприятий".

Варианты отчета 

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Доступны два предопределенных варианта отчета: 

  1. История изменения дат ремонта - открывается по умолчанию. Данные о ремонтах сгруппированы по объектам ремонта в разрезе вида ремонта. Изменение плановых дат ремонтов регистрируются документами "Смета ремонта (Заявка на ремонт)" для документов "Выявленные дефекты", "Внешнее основание для работ", "План-график ППР"; документами "Закрытие заявок и ремонтов", "Остановочный ремонт" - для документа "план-график ППР"; документом "План работ" - для документов "План-график ППР", "График регламентных мероприятий":
    • Объект ремонта - наименование объекта ремонта;
    • Вид ремонта - вид ремонта (поле "Вид ремонта" из документа "План-график ППР"; поле "Вид мероприятия" из документа "График регламентных мероприятий"; поля "Вид ремонта при вводе на основании выявленных дефектов" и "Вид ремонта при вводе на основании внешних оснований" из настроек параметров подсистемы ТОИР);
    • Дата изменения - дата регистрации документа-основания в системе (поле "от" из шапки документов);
    • Документ-основание - ссылка на документы: "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР", "График регламентных мероприятий", "Смета ремонта "Заявка на ремонт", "План работ", "Закрытие заявок и ремонтов", "Остановочные ремонты");
    • Плановая дата начала - планируемая дата начала ремонтных работ (поле "Дата начала" из документа "Смета ремонта (Заявка на ремонт)" для документов "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР"; поле "Период" из документов "План-график ППР", "График регламентных мероприятий" для документов "План-график ППР" и "График регламентных мероприятий"; поле "Начало" группировки "Текущие даты" из документа "Остановочный ремонт" и поле "Новая дата начала" из документа "Закрытие заявок и ремонтов" для документа "План-график ППР"; поле "Дата начала" группировки "Текущие даты" из документа "План работ" для документов "План-график ППР", "График регламентных мероприятий");
    • Плановая дата окончания - планируемая дата окончания ремонтных работ (поле "Дата окончания" из документа "Смета ремонта (Заявка на ремонт") для документов "Внешнее основание для работ", "Выявленные дефекты", "План-график ППР"; поле "Период" из документов "План-график ППР", "График регламентных мероприятий" для документов "План-график ППР" и "График регламентных мероприятий"; поле "Окончание" группировки "Текущие даты" из документа "Остановочный ремонт" и поле "Новая дата окончания" из документа "Закрытие заявок и ремонтов" для документа "План-график ППР"; поле "Дата окончания" группировки "Текущие даты" из документа "План работ" для документов "План-график ППР", "График регламентных мероприятий").
  2. История изменения дат ремонта (с иерархией) - вариант отчета, в котором материальные затраты ремонтов сгруппированы по объекту иерархии, объекту ремонта:
    • Объект иерархии - наименования всех родителей объекта ремонта (объекты ремонта или группы объектов ремонта).
Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
