Документ «Анализ причин дефекта»

Документ предназначен для анализа причин возникновения дефекта оборудования посредством метода "5 почему". 

Для использования анализа причин дефекта необходимо включить функциональную опцию "Использовать анализ причин дефектов".

В форме выводятся следующие поля:

  • Номер – номер документа в системе.
  • От – дата регистрации документа в системе.
  • Организация – наименование организации, проводящей анализ причины дефекта.
  • Подразделение – наименование подразделения, проводящего анализ причины дефекта.
  • Документ-основание – ссылка на документ "Выявленные дефекты", на основании которого создан документ.
  • Статус документа — текущий статус документа. Выбирается из предопределенного списка и может принимать следующие значения:
    • Зарегистрирован. Статус по умолчанию.
    • В работе. Устанавливается пользователем вручную или автоматически (при заполнении табличной части "Дерево анализа").
    • Анализ выполнен. Устанавливается пользователем вручную после заполнения реквизитов документа, а также коренных и непосредственной причин. Документ становится недоступен для редактирования, за исключением таблицы "Корректирующие мероприятия".
    • Закрыт. Устанавливается пользователем вручную. Выполняется проверка на заполненность ТЧ "Дерево анализа" и ТЧ "Корректирующие мероприятия". По корректирующим мероприятиям ремонты должны быть выполнены (или отменены) и завершены все "Задания" бизнес-процессов.
  • Дата и время события – дата обнаружения выявленного дефекта.
  • Объект ремонта – наименование объекта ремонта, для которого выявляется коренная причина возникновения дефекта.
  • Дата завершения анализа – дата завершения анализа причин дефекта. Заполняется автоматически при переводе документа в статус "Анализ выполнен" либо, если этот статус был пропущен, в статус "Закрыт".
  • Комментарий – дополнительная текстовая информация об анализе.
  • Ответственный – ФИО пользователя, ответственного за анализ.
  • Автор – ФИО пользователя, создавшего документ. Недоступен для редактирования.

Закладка «Описание события»

  • Краткое описание — краткое описание выявленного дефекта. По умолчанию заполняется из поля "Описание дефекта" документа "Выявленные дефекты".
  • Описание — подробное описание дефекта.

Закладка «Анализ причин»

  • Комиссия — комиссия, ответственная за проведение анализа причин дефектов. Становится доступно, если включена функциональная опция "Использовать комиссии".
  • Причина/заключение — непосредственная причина, промежуточные причины, заключения или коренные причины возникновения дефектов.
  • Это заключение — флаг, устанавливается автоматически для заключений.

Кнопка [Добавить причину] предназначена для добавления причин выявленного дефекта.

Кнопка [Добавить заключение] предназначена для добавления заключений для выявленного дефекта.

Закладка «Корректирующие мероприятия»

  • N — порядковый номер записи в таблице.
  • Описание — дополнительная текстовая информация, заполняемая из описания корректирующего мероприятия.
  • Ответственный — ФИО сотрудника, ответственного за выполнение мероприятия.
  • Срок — плановый срок выполнения работ.
  • Внешнее основание/Задание — ссылка на документ "Внешнее основание" или бизнес процесс "Задание"*.
  • Статус — статус ремонта (если по документу "Внешнее основание для работ" введен документ "Смета ремонта (Заявка на ремонт)"), либо документа "Внешнее основание для работ" (при отсутствии "Сметы ремонта (Заявки на ремонт)"), либо состояние бизнес-процесса.

Кнопка [Создать внешнее основание] предназначена добавления внешних оснований.

Кнопка [Назначить задания] открывает форму, на которой пользователь должен установить, для каких именно строк табличной части он хочет назначить задачи.

Примечания:

Для работы с "Заданиями" необходимо включить функциональную опцию "Использовать бизнес процессы и задания" (Настройка и администрирование – Органайзер - Бизнес-процессы и задачи).
