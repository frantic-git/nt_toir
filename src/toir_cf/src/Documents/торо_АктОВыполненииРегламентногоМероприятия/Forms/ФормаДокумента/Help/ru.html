Документ «Акт о выполнении регламентного мероприятия»

Документ предназначен для отражения факта выполнения подразделением регламентных мероприятий.

В форме документа выводятся следующие поля:

  • Номер – номер документа.
  • От – дата регистрации документа.
  • Организация – наименование организации, от имени которой оформляется акт о выполнении регламентного мероприятия.
  • Подразделение –  наименование подразделения, которое фиксирует факт выполнения регламентного мероприятия.
  • Завершить мероприятия – признак, указывающий на то, что выполнение заявленных ранее мероприятий завершено.
  • Ответственный – ФИО пользователя, создавшего акт.
  • Комментарий – дополнительная текстовая информация об акте.
  • Автор – ФИО пользователя, создавшего документ. Недоступен для редактирования.

Закладка «Ремонтные работы»

Содержит информацию о списке объектов регламентных мероприятий и видах мероприятий, выполненных в рамках акта.
  • N – порядковый номер записи в списке.
  • Список объектов – наименование списка объектов регламентных мероприятий, на выполнение мероприятий по которым оформлен акт.
  • Вид мероприятия – вид регламентного мероприятия.
  • Дата начала – фактическая дата и время начала ремонта.
  • Дата окончания – фактическая дата и время окончания ремонта.
  • Завершить мероприятие – признак, указывающий завершение ремонтных работ по данному объекту ремонта.

Кнопка [Подбор] предназначена для оперативного формирования списка с помощью обработки «Подбор работ по ТОиР из документов-оснований».

Подзакладка «Ремонтные работы»

Имеет иерархическую структуру и содержит перечень ремонтных работ и их технологических операций для списка объектов регламентных мероприятий.
  • Ремонтная работа – название работы, по которой проходит акт о выполнении регламентного мероприятия.
  • Количество –  количество ремонтов или технологических операций, закрываемых актом.
  • Ед. изм. – единица измерения операции.
  • Выполнено – признак выполнения ремонтной работы.
  • Выполнение, % – процентное значение количества выполненных работ. Становится доступным при включении функциональной опции "Использовать усложненную схему закрытия нарядов" в разделе Настройка параметров системы - Интерфейс и ввод документов - Наряды и работы.
  • Новая операция – признак отсутствия ремонтной работы в документе-основании.
Кнопка [Подбор] предназначена для оперативного формирования списка технологических операций.
Кнопка [Подбор по бригадам] позволяет подобрать исполнителей из сформированных бригад в соответствии с требуемыми квалификациями исполнителей.

Подзакладка «Список объектов»

Содержит список объектов регламентного мероприятия.
  • Объект ремонта – название объекта ремонта, входящего в список объектов регламентного мероприятия. 
  • Выполнено –  признак выполнения работы по отдельному объекту ремонта.
  • Комментарий – текстовый комментарий.

Подзакладка «Материальные затраты»

Содержит список номенклатуры, которая была затрачена при проведении регламентного мероприятия. Сведения в списке отображаются в разрезе выбранной технологической операции.
  •   - признак, указывающий наличие у данной номенклатуры аналога. Признак становится доступен при включении функциональной опции "Использовать аналоги номенклатуры" в разделе Настройка параметров системы - Интерфейс и ввод документов - МТО.
  • Номенклатура – наименование номенклатуры.
  • Ед. изм. – единица измерения номенклатуры.
  • Количество – количество номенклатуры.

Подзакладка «Исполнители»

Содержит список исполнителей ремонтных работ с возможностью указания фактической суммы к начислению. Исполнителями могут выступать сотрудники ремонтного предприятия, сторонние контрагенты или те и другие одновременно.

Подзакладка «Хозспособ»

Содержит список квалификаций специалистов с указанием количества и необходимого времени для выполнения тех. операций ремонтов.
  • Сотрудник – сотрудник, который является исполнителем тех.операции.
  • Квалификация – квалификация сотрудника.
  • Дата начала – дата и время начала ремонтных работ.
  • Дата окончания – дата и время окончания ремонтных работ.
  • Время работы (на 1 объект) – длительность выполнения работы над одним объектом регламентного мероприятия.
  •   – признак, при установке которого время будет рассчитываться автоматически исходя из даты и времени начала и окончания работ.
  • КТУ – коэффициент трудового участия сотрудника.
  • Стоимость работ – сумма для начисления сотруднику за выполнение ремонтной операции.

Кнопка [Добавить] позволяет добавить исполнителя тех. операции.

Кнопка [Заполнить] открывает выпадающее меню, в котором можно выбрать следующие пункты:

Кнопка [Заполнить исполнителей] предназначена для подбора исполнителей в документ.

Кнопка [Заполнить стоимости] предназначена для заполнения суммы начисления сотруднику за выполнение ремонтной операции одним из следующих способов: По тарифной ставке сотрудника, По стоимости часа квалификации, По расценке тех. операции.

Кнопка [Заполнить время работы] предназначена для заполнения времени выполнения работы сотрудником одним из следующих способов: По нормативам, По длительности ремонтных работ с учетом графика работы сотрудника.

Кнопка [Заполнить даты работ] предназначена для заполнения дат и времени начала и окончания ремонтных работ по каждому сотруднику, используя дату и время начала и окончания ремонта.

Подзакладка «Подрядный»

Содержит список подрядчиков, с указанием даты выполнения мероприятий, договора и стоимости работ.
  • Контрагент – наименование контрагента, выполнившего регламентное мероприятие.
  • Договор контрагента – наименование договора, по которому выполнялись работы.
  • Дата начала – фактическая дата начала мероприятия.
  • Дата окончания – фактическая дата окончания мероприятия.
  • Стоимость работ – сумма, на которую было проведено мероприятие.
Кнопка [Заполнить] открывает выпадающее меню, в котором можно заполнить даты работ по ремонтам.

Закладка «Документы-основания»

Содержит список документов, на основании которых создан акт (график регламентного мероприятия или наряд на регламентное мероприятие). 
  • N – номер элемента списка.
  • Документ-основание – наименование, номер и дата документа-основания.

Закладка «Показатели, наработка, состояния»

Становится доступна при включении хотя бы одной из функциональных опций "Учет контролируемых показателей", "Учет наработки оборудования" или "Учет состояния оборудования" в разделе Настройка параметров системы - Интерфейс и ввод документов - Показатели эксплуатации.
Предназначена для введения наработки, значений контролируемых показателей и состояний объектов ремонта, участвовавших в регламентном мероприятии. Чтобы в акте можно было ввести необходимые документы учета, необходимо указать их на подзакладке "Выходные документы" списка объектов регламентных мероприятий, который участвует в акте.
  • Документ – наименование, номер и дата документа.
Добавление документов осуществляется при нажатии на кнопку [Добавить]. Добавленные документы группируются в разворачиваемые группы по типу введенных данных.

Закладка «Дополнительно»

Данная закладка содержит в себе поле выбора комиссии, члены которой будут вписаны в печатную форма акта о выполнении регламентного мероприятия. Становится доступна, если установлен признак "Использовать комиссии" в разделе Настройка параметров системы - Интерфейс и ввод документов - Наряды и работы.
