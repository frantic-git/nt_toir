Документ «Состояния объектов ремонта»

Документ предназначен для ввода информации о простоях и испытаниях оборудования.

В форме документа выводятся следующие поля:

  • Номер – номер документа в системе.
  • От – дата регистрации документа в системе.
  • Организация – организация, от имени которой оформляется документ.
  • Подразделение – подразделение, в котором оформляется документ.
  • Вид операции – вид операции ввода состояния: состояние на дату или на период.
  • Вид эксплуатации на период  – вид эксплуатации на дату или в течение периода, на который фиксируется состояние, значение поля подставляется в табличную часть в одноименное поле.
  • Комментарий – текстовый комментарий к документу.
  • Ответственный – ФИО пользователя, ответственного за документ.
  • Автор – ФИО пользователя, создавшего документ. Недоступен для редактирования.

Табличная часть «Объекты ремонта»


При вводе состояния на дату в табличной части отображаются следующие поля:
  • N – порядковый номер записи в списке.
  • Положение – положение объекта ремонта в структуре иерархии.
  • Объект ремонта – наименование объекта ремонта, для которого регистрируется состояние.
  • Распространять на подчиненные ОР – отметка, позволяющая распространить состояние объекта ремонта на подчиненные ему объекты ремонта.
  • Структура иерархии – структура иерархии, в которой рассматривается данный объект ремонта.
  • Дата изменения состояния – дата, на которую необходимо ввести изменение состояния оборудования.

  • Вид эксплуатации /Вид эксплуатации на период – вид эксплуатации объекта ремонта на дату / в рамках периода.

  • Причина простоя – отображается, если выбран вид эксплуатации "Простой"; в поле необходимо ввести причину.

При вводе состояния на период в табличной части отображаются следующие поля:
  • N – порядковый номер записи в списке.
  • Положение – положение объекта ремонта в структуре иерархии.
  • Объект ремонта – наименование объекта ремонта, для которого регистрируется состояние.
  • Распространять на подчиненные ОР – отметка, позволяющая распространить состояние объекта ремонта на подчиненные ему объекты ремонта.
  • Структура иерархии – структура иерархии, в которой рассматривается данный объект ремонта.

  • Начало периода – дата начала периода, на который вводится состояние.

  • Вид эксплуатации на период – вид эксплуатации объекта ремонта в рамках периода.

  • Окончание периода – дата окончания периода, на который вводится состояние.

  • Вид эксплуатации – вид эксплуатации объекта ремонта по окончании периода.

  • Причина простоя – отображается, если выбран вид эксплуатации "Простой"; в поле необходимо ввести причину.
Кнопка  [Подбор] предназначена для оперативного формирования списка объектов ремонта с помощью формы подбора.
С помощью кнопки  [Заполнить по умолчанию] осуществляется заполнение состояний объектов ремонтов в табличной части на основании уже имеющихся в системе документов по состоянию оборудования.
