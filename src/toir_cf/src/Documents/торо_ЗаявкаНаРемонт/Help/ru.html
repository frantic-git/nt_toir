Документ «Смета ремонта (Заявка на ремонт)»

Документ предназначен для регистрации намерения проведения ремонтных работ на указанный период для объекта ремонта.

Заявкой на ремонт пользователь планирует на определенную дату выполнение работ по ремонту и обслуживанию, указывая перечень работ, а также сведения о необходимых затратах для их проведения. 

Заявка вводится на основании выявленных дефектов, внешнего основания для работ или графика ППР. В первом случае основанием служит документ, а во втором – строка ремонтных работ из графика ППР.

В форме выводятся следующие поля:

  • Номер – номер документа.
  • от – дата регистрации заявки в системе.
  • Организация – наименование организации, инициирующей заявку.
  • Подразделение – наименование подразделения, инициирующей заявку. По значению данных полей накладывается отбор при подборе ремонтов в документ (можно выбрать только объекты, принадлежащие указанным Организации и Подразделению).
  • Ответственный – ФИО пользователя, ответственного за оформление заявки.
  • Комментарий – дополнительная текстовая информация о заявке на ремонт.
  • Автор – ФИО пользователя, создавшего документ. Недоступен для редактирования.

Закладка «Ремонтные работы»

В верхней табличной части выводятся следующие поля:
  • N – порядковый номер записи в списке.
  • Объект ремонта – наименование объекта ремонта, для которого формируется заявка на ремонт.
  • Вид ремонтных работ – наименование вида работ по ремонту и обслуживанию оборудования, выполняемых в рамках текущего документа.
  • Дата начала – планируемая дата начала ремонта.
  • Дата окончания – планируемая дата окончания ремонта.
  • Актуальная дата начала – актуальная дата начала ремонта.
  • Актуальная дата окончания – актуальная дата окончания ремонта.
  • Крайняя дата завершения – отображается при включенной функциональной опции "Назначение на ремонт с учетом рисков". Дата начала ремонта, выбранная с помощью обработки "Матрица оценки рисков".
  • Актуальная крайняя дата завершения – отображается при включенной функциональной опции "Назначение на ремонт с учетом рисков". Дата начала ремонта, выбранная с помощью обработки "Матрица оценки рисков".
  • Описание ремонта – текстовое описание ремонта.
  • Критичность дефекта – заполняется для сметы на основании выявленного дефекта с указанной критичностью дефекта. 
  • Г – признак того, что оборудование на гарантии.
  • В рамках остановочного – отметка, что ремонт выполняется в рамках остановочного.
  • Статус ремонта – статус, в котором находится заявка.
Кнопка [Подбор] предназначена для оперативного формирования списка ремонтов из документов-оснований.
Кнопка [Фильтр] позволяет отфильтровать ремонты по исполнителям.
Кнопка [История изменения дат ремонта] открывает одноименный отчет.
Кнопка [Открыть матрицу оценки рисков] открывает одноименную обработку.

Заполнить продолжительность выполнения работ по технологической карте ремонта можно при нажатии кнопки [Продолжительность] одним из следующих способов:
  • По нормам времени тех. операций – суммируется длительность выполнения каждой тех. операции, указанной в списке операций ремонта.
  • По продолжительности трудозатрат – суммируется время работы на закладке "Трудозатраты" для каждой тех. операции, указанной в списке операций ремонта, с учетом значения в поле "Количество".
  • По продолжительности операций ремонтов – аналогично как и по нормам времени тех. операций, но в данном случае значения будут браться не из элементов, на которые ссылаются записи в списке операций ремонта, а непосредственно из списка в поле "Продолжительность ".
  • Ввести продолжительность — ввод продолжительности вручную.
  • Ввести продолжительность с учетом графика – ввод продолжительности с учётом графика работы оборудования. 
В левой табличной части выводятся следующие поля:
  • Ремонтная работа – наименование ремонтной работы.
  • Кол-во – количество единиц ремонтной работы.
  • Ед. изм. – единица измерения ремонтных работ.
  • Исполнитель – исполнитель ремонтной работы.
  • Договор/бригада – договор, по которому будет выполняться ремонтная работа, или бригада, которая будет ее выполнять. 
Кнопка [Подбор] предназначена для оперативного формирования списка ремонтных работ с помощью формы подбора.

Подзакладка «Материальные затраты»

  •   – признак, указывающий наличие у данной номенклатуры аналога. Признак становится доступен при включении функциональной опции "Использовать аналоги номенклатуры" в разделе Настройка параметров системы - Интерфейс и ввод документов - МТО.
  • Номенклатура – наименование номенклатуры, необходимой для выполнения выбранной технологической операции.
  • Характеристика номенклатуры – характеристика номенклатуры, необходимой для выполнения выбранной технологической операции.
  • Ед. изм. – наименование единицы измерения, в которой исчисляется количество затраченной номенклатуры.
  • Количество единиц – необходимое количество номенклатуры в заданных единицах измерения.

Кнопка [Подбор] предназначена для оперативного формирования списка материальных затрат с помощью  обработки "Подбор номенклатуры".

Табличная часть «Заказы на внутреннее потребление»

  • Заказ на внутреннее потребление – дата и номер заказа на внутреннее потребление.
  • Дата отгрузки – планируемая дата получения заказа.
  • Склад – предпочтительный склад получения товара.
  • Обеспечен – отметка, что заказ на внутреннее потребление обеспечен.
Кнопка [Добавить заказ на внутреннее потребление] позволяет добавить заказ на внутреннее потребление.
Кнопка [Сводный документ заказ на внутреннее потребление] предназначена для добавления сводного заказа.

Подзакладка «Инструменты и техника»
  • Инструмент – наименование позиции номенклатуры, которая будет учитываться в качестве инструмента при выполнении заявленных ремонтах работ.
  • Ед. изм. – наименование единицы измерения, в которой исчисляется количество номенклатуры указанной позиции.
  • Количество – количество номенклатуры указанной позиции.

Кнопка [Подбор] предназначена для оперативного формирования списка с помощью обработки "Подбор номенклатуры".

Подзакладка «Запчасти»

  • Номенклатура – наименование номенклатуры, необходимой для выполнения выбранной технологической операции. 
  • Характеристика номенклатуры – характеристика номенклатуры, необходимой для выполнения выбранной технологической операции. Отображается при включенной функциональной опции "Использовать характеристики номенклатуры".
  • Ед. изм. – наименование единицы измерения, в которой исчисляется количество затраченной номенклатуры. 
  • Количество – необходимое количество номенклатуры в заданных единицах измерения. 

Кнопка [Подбор] предназначена для оперативного формирования списка с помощью обработки "Подбор номенклатуры".

Подзакладка «Исполнители/По ремонту»

  • Исполнитель — наименование подразделения-исполнителя ремонтных работ.
  • Договор / Бригада – договор, по которому будет выполняться ремонтная работа, или бригада, которая будет ее выполнять. 
  • Статус ремонта – статус ремонта по заявке.
  • Комментарий к статусу – комментарий, поясняющий статус.
Кнопка [Заполнить из лотов] предназначена для автоматического заполнения исполнителя подрядчиками из лотов. Активна при включенной функциональной опции "Использовать лоты".
Кнопка [Отказ исполнителя] позволяет отказаться от выполнения ремонтной работы.

Подзакладка «Исполнители/По операциям» 
  • Квалификация — наименование квалификации исполнителя заявленных ремонтных работ.
  • Время работы – продолжительность работы указанного количества сотрудников.
  • Количество – количество единиц сотрудников указанной квалификации.

Кнопка [Заполнить из лотов] предназначена для автоматического заполнения исполнителя подрядчиками из лотов.

Подзакладка «Меры безопасности»

  • N – порядковый номер записи в списке.
  • Меры безопасности – краткое название свода мер безопасности, которые необходимо соблюдать при выполнении запланированных ремонтных работ.

Закладка «Документы-основания»

Таблица «Документы-основания»

  • N – порядковый номер записи в списке.
  • Документ-основание – наименование, номер и дата документа, на основании которого сформирована заявка.

Таблица «Закрываемые предписания»

  • Объект ремонта – наименование объекта ремонта. 
  • Плановая дата ремонта – плановая дата работы по внешнему основанию. 
  • Документ внешнего основания – дата и номер документа внешнего основания.
  • Описание – дополнительная текстовая информация о внешнем основании для работ.
