Справочник «Цепочки ремонтов»

Справочник предназначен для ведения списка фиксированных ремонтных циклов.

В форме элемента справочника выводятся следующие поля:
  • Группа — наименование группы, в которую входит цепочка, может быть не заполнено.
  • Код — код цепочки ремонтов.
  • Наименование — название цепочки ремонтов.
  • Способ учеты дата отсчета — название способа, в соответствии с которым учитывается дата отсчета графика ППР.

Табличная часть «Последовательность ремонтов»

  • N — порядковый номер записи в списке.
  • Вид ремонта — вид ремонта в последовательности.
  • Технологическая карта – наименование технологической карты ремонта.
  • Способ планирования — наименование способа планирования, который будет использоваться при расчете графика ППР для определенного вида ремонта.
  • График ремонтных работ — наименование графика работы ремонтного персонала, в соответствии с которым производится планирование ТО и ремонтов
  • Способ выполнения — наименование способа выполнения ремонтных работ: хозспособ или подрядный.
  • СМИ — признак того, что запланированный ремонт можно перенести в сторону сокращения межремонтного периода, если он попал на выходные. Если перенесенная дата ремонта попадает в архивный период, перенос не делается. Если отметка не установлена и для этого вида ремонта ремонты на выходных не выполняются, то при попадании следующих плановых ремонтов на выходные, начало ремонта переносится на первый рабочий день после выходных.
  • Не учитывать ГРО — признак, определяющий способ расчета плановой наработки для определения даты следующего ремонта при расчете графика ППР. Если признак не установлен, то расчет плановой наработки будет выполняться с учетом графика работы оборудования, т.е. если оборудование не работает в выходные, считается, что у него в выходные наработки нет, а в остальные дни наработка будет равна указанной для объекта ремонта на закладке "Наработка". Если признак установлен, то расчет плановой наработки будет выполняться без учета графика работы оборудования, и наработка за любой день будет одинакова и равна указанной для объекта ремонта на закладке "Наработка".
  • Не учитывать ГРР — признак того, что при формировании графика ППР ремонты будут запланированы без учета графика ремонтов оборудования, заданного в ремонтной группе объекта. Т.е. если ремонты согласно графику ремонтных работ в ремонтной группе не выполняются в выходные, то ремонты на выходные все равно будут запланированы – длительность ремонтов будет календарной.
  • Не учитывать продолжительность — признак, определяющий дату начала планирования следующего ремонта. Если признак установлен, то в качестве даты начала следующего ремонта определенного вида будет использована дата окончания предыдущего. Если признак не установлен, то в качестве даты начала следующего ремонта определенного вида будет использована дата дня, следующего за окончанием предыдущего.
