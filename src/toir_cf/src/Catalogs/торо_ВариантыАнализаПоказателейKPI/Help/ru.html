Справочник «Варианты анализа показателей»

Подчинен справочнику "Показатели процессов (KPI)", определяет внешний вид значений показателя в мониторе показателей (KPI).

В форме элемента справочника выводятся следующие поля:

  • Наименование варианта – название варианта анализа показателей.
  • Показатель – название показателя, который анализируется в варианте.
  • Ед. изм. – единица измерения показателя.
  • Тип анализа – тип анализа в варианте анализ; ключевой реквизит справочника, от него зависит доступность остальных настроек. Доступные варианты:
    • Динамика изменения. Этот тип анализа применяется в том случае, если необходимо проанализировать изменения показателя за определенный период. Данные показателя будут получены с разбивкой по периодичности, указанной в настройках монитора. После этого таблица сворачивается, в ней остаются только колонки "Период" и "Значение", без аналитик.
    • Покомпонентное сравнение (структура).  Этот тип анализа применяется в том случае, если необходимо произвести сравнительный анализ данных по объектам анализа (организация, подразделение, объект ремонта). Данные показателя будут получены без разбивки по периодичности и свернуты по аналитике.
    • Покомпонентное сравнение (динамика). Этот тип анализа применяется в том случае, если необходимо произвести сравнительный анализ изменения данных с течением времени по объектам анализа (организация, подразделение, объект ремонта). Данные показателя будут получены с разбивкой по периодичности, указанной в настройках монитора, и свернуты по периоду и аналитике.
    • Сравнение с прошлым периодом. Этот тип анализа применяется в том случае, если необходимо сравнить состояние показателя в текущем периоде с данными предыдущего периода (предыдущий месяц, аналогичный период прошлого года и т.д.). Данные показателя будут получены без разбивки по периодичности, но за два периода, указанных в настройках монитора. Обе таблицы сворачиваются до одного числового значения. Два полученных значения выводятся в монитор.
    • Измерение текущего состояния. Данный тип анализа показывает индикатор текущего состояния показателя.  Данные показателя будут получены без разбивки по периодичности и свернуты до одного числового значения.
  • Значение – название поля числового типа, которое  будет выводиться в монитор, т.к. в источнике данных показателя может быть несколько таких полей.
  • Агрегация данных – агрегатная функция, которая будет использоваться при сворачивании. Если выбран один из типов анализа "Покомпонентное сравнение…", то после сворачивания и агрегации к таблице данных можно применить сортировку или ограничить объем выводимых данных, отобрав заданное количество объектов анализа с наибольшими или наименьшими значениями показателя (реквизит "Выбрать сортировку").
  • Выбрать сортировку – выбор варианта сортировки выводимых данных.
  • Выполнить нормализацию – настройка, при выборе которой появляется возможность нормировать значения показателя. Значения показателя рассчитываются как отклонение значения от целевого значения показателя (для целевого тренда "удержание в пределах допустимого диапазона" - от центра целевого диапазона) по формуле "(ЗначениеПоказателя – ЦелевоеЗначение) / ЦелевоеЗначение * 100%", где ЦелевоеЗначение– целевое значения показателя (центр целевого диапазона), введенный документом "Установка целевых значений показателей (KPI)".
  • Дробная часть – выбор количества знаков после запятой при округлении значений показателя.
  • Вариант отображения – форма представления данных в мониторе:
    • Кратко – вариант доступен только для типов анализа "Сравнение с прошлым периодом", "Измерение текущего состояния". Значение показателя выводится в компактной текстовой форме.
    • Диаграмма – данные показателя выводятся в диаграмму. Доступные типы диаграмм определяются типом анализа.
    • Таблица – данные показателя выводятся в таблицу с колонками "Период" (если есть), "Разрез" (если есть), "Значение".
  • Тип диаграммы – тип диаграммы, если в качестве варианта отображения выбрана она.
  • Отображать нулевые значения – настройка, позволяющая отображать нулевые значения.
