Обработка «Общий план работ»

Обработка предназначена для вывода в виде диаграммы или таблица информации о ремонтах по объектам ремонта, их владельцам, видам ремонта и исполнителям на различных временных промежутках с различной детализацией, а также для распределения дат ремонтных работ. 

Отображение обработки зависит от функциональный опции "Использовать общий план работ", расположенной в разделе Настройка параметров системы - Интерфейс и ввод документов - Планирование ТОиР.

Кнопка [План работ] формирует печатную форму документа "Общий план работ".

Кнопка [План МТО] открывает отчет "План МТО".

Диаграмма и таблица ремонтов формируются с помощью кнопки [Построить/Обновить].

Кнопка [Сохранить план] позволяет сохранить итоговый план работ подразделения с датами и становится недоступна, если дата отсчета в обработке больше даты отсчета в выбранном плане или если в статусе согласования запрещено редактирование.

На форме выводятся следующие поля:

  • Организация – организация для отбора.
  • Вид плана – вид скользящего плана для отбора. При выборе плана могут изменяться значения в других группах отбора.
  • Дата отсчета – дата отсчета, по которой будет произведен отбор.

Группа «Выводить информацию по»

  • План-графикам ППР – отбор, позволяющий выводить ремонты из документов  "План-график ППР".
  • Графикам регламентных мероприятий - отбор, позволяющий выводить ремонты из документов  "График регламентных мероприятий".
  • Заявкам по дефектам – отбор, позволяющий выводить ремонты из документов "Выявленные дефекты".
  • Заявкам по внешним основаниям – отбор, позволяющий выводить ремонты из документов "Внешнее основание для работ".

Группа «Глубина планирования»

  • Количество периодов – число выводимых периодов на диаграмме.
  • Периодичность – период, по которому будут отображаться работы.
  • Детализация – детализация в рамках периода.
  • Вид исполнителя работ - вид исполнителя работ, выводимых на диаграмме.

Группа «Настройка отображения»

Позволяет настроить внешний вид данных в табличной части и на диаграмме.

  • Верхний уровень – группировка для верхнего уровня.
  • Нижний уровень – группировка для нижнего уровня.

Переключатель [Группировать по] позволяет выбрать группировку по исполнителю ремонта или по владельцу объекта ремонта.

Группа «Отборы по объектам и видам ремонта»

  • Владелец ОР – владелец объекта ремонта для отбора.
  • Объект ремонта – объект ремонта для отбора.
  • Вид ремонта – вид ремонта для отбора.

Группа «Отбор по исполнителям»

  • Подразделение-исполнитель – подразделение-исполнитель для отбора.
  • Бригада – бригада, назначенная на ремонт.
  • Контрагент – контрагент, выполняющий ремонт.
  • Договор контрагента – договор, по которому контрагент выполняет ремонт.

Кнопка [Построить/Обновить] обновляет табличную часть и диаграмму.

Кнопка [Незапланированные ремонты] отображается рядом с кнопкой [Построить/Обновить], если выбран определенный план работ, и позволяет отобразить незапланированные ремонты.

Гиперссылка Выбрать план работ позволяет выбрать план работ среди имеющихся планов в системе. При выборе плана заполненные данные могут измениться в соответствии с планом. Если план выбран, то гиперссылка станет открывать текущий план работ. 

Выбранный план можно отменить, нажав на крестик рядом с гиперссылкой. Отображение гиперссылки зависит от функциональной опции "Использовать скользящее планирование", расположенной в разделе Настройка параметров системы - Интерфейс и ввод документов - Планирование ТОиР.

Закладка «Таблица ремонтов»

  • Исполнитель (Подразделение, контрагент)/Владелец ОР) – исполнитель ремонтных работ либо владелец объекта ремонта (иерархия настраивается в группе Настройка отображения).
  • Критичность ОР – критичность ОР.
  • Критичность дефекта – критичность дефекта.
  • Плановые даты – плановые даты ремонта. Группировка состоит из следующих полей:
    • Дата начала – дата начала планового ремонта.
    • Дата окончания – дата окончания планового ремонта.
  • Редактируемые даты – редактируемые даты ремонта. Группировка состоит из следующих полей:
    • Дата начала – дата начала ремонта (редактируемая).
    • Дата окончания – дата окончания ремонта (редактируемая).
  • Документ-основание – наименование документа-основания для ремонта.
  • План работ – ссылка на документ "План работ".
  • Смета ремонта (заявка на ремонт) – отметка о наличии у ремонта сметы (заявки на ремонт).
  • Наряд на выполнение ремонтных работ – отметка о наличии у ремонта наряда.
  • Акт о выполнении этапа работ – отметка о наличии у ремонта акта. 
  • Остановочный ремонт – отметка о том, что ремонт является остановочным.

Группа «Распределение дат по графику»

Отображается на закладке "Таблица ремонтов" при включенной функциональной опции "Использовать скользящее планирование", расположенной в разделе Настройка параметров системы - Интерфейс и ввод документов - Планирование ТОиР и предназначенной для распределения дат ремонтов.

  • Дата начала расчета – дата, начиная с которой будут распределены ремонты.
  • График работы – график работы, по которому будут распределены ремонты.
Кнопка [Заполнить] позволяет распределить даты либо для всех ремонтов в табличной части либо для выбранных ремонтов.

Закладка «Диаграмма ремонтов»

Содержит диаграмму Ганта, построенную на основании отобранных пользователем данных.
