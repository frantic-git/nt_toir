Режим «Настройка параметров системы»

Режим предназначен для настройки параметров функционирования системы.  Настройки являются глобальными, т.е. настраиваемыми для всей системы в целом.

Закладка «Общие»

  • Валюта управленческого учета — название валюты, в которой будет указываться сумма материальных и трудовых затрат на выполнение работ по ремонту и ТО в отчетах системы.
  • Вид ремонта при вводе на основании выявленных дефектов — название вида работ по обслуживанию ОС, который используется по умолчанию при формировании документов на основании документа "Выявленные дефекты".
  • Вид ремонта при вводе на основании внешних оснований — название вида работ по обслуживанию ОС, который используется по умолчанию при формировании документов на основании документа "Внешнее основание для работ".
  • Технологическая карта при вводе на основании выявленных дефектов — используется, если: 1) в типовом дефекте не указана тех.карта (либо в документе не указан сам типовой дефект); 2) для объекта ремонта не настроен нормативный ремонт для вида ремонта из настройки "Вид ремонта при вводе на основании выявленных дефектов".
  • Иерархия для ввода новых объектов ремонта — структура иерархии объектов ремонта предприятия, используемая для ввода новых ОР.
  • График ремонтных работ — название графика работ, который будет подставляться по умолчанию для нормативных ремонтов в справочнике Ремонтные группы в том случае, если в режиме "Настройки пользователя" параметр "График работы" не указан. 
  • Количество дней смещения в дефекте для простоев — период, в течение которого не осуществляется автоматическая регистрация состояний "Простой" на основании выявленных дефектов.
  • Вид эксплуатации для ввода на основании выявленного дефекта — название вида эксплуатации, который используется по умолчанию при формировании документа "Состояние ОР" на основании документа "Выявленные дефекты". 
  • Причина закрытия ремонтов — причина досрочного закрытия ремонтов, подставляемая по умолчанию в документе "Закрытие заявок и ремонтов".
  • Причина закрытия заявок — причина досрочного закрытия неисполненных заявок. 
  • Тип цены для расчета плановой себестоимости ремонта — название типа цен номенклатуры, в соответствии с которым в отчетах системы рассчитываются плановые суммы затрат МТО. 
  • Тип цены для расчета фактической себестоимости ремонта — тип цены номенклатуры, в соответствии с которым в отчетах системы рассчитываются фактические суммы затрат МТО.
  • Наименование ОР для печати — представление наименования объектов ремонта, который будет использоваться в печатных формах: наименование, наименование и код, полное наименование объекта ремонта, наименование и инвентарный номер.
  • Представление вида ремонта для печати — представление наименования вида ремонта, который будет использоваться в печатных формах: наименование, код, наименовании и код. 
  • Представление вида ремонта в структуре ремонтного цикла — представление наименования вида ремонта, которое будет использоваться в структуре ремонтного цикла. 
  • Версия ППР — информация о текущей версии обработки ППР.
  • Таблица соответствия версий обработок версиям релиза — информация по соответствию текущей версии обработки ППР текущей версии системы.

Закладка «Интерфейс и ввод документов»

  • Выбранный сценарий работы — выбор шаблона сценария работы системы.
  • Настройка интерфейса и используемого функционала — закладка из 7 подзакладок (Общие, Оборудование и нормативы, Показатели эксплуатации, Планирование ТОиР, Наряды и работы, МТО, Интеграции). Установка определенных флагов на каждой из закладок открывает доступ к соответствующему функционалу системы или активирует соответствующую закладку.
  • Настройка бизнес-процессов — группа закладок, на которых с помощью установки соответствующих флажков определяется необходимость введения и последовательность документов по тому или иному бизнес-процессу.

Подзакладка «Общие»

  • Использовать согласование — включает возможность согласования документов, активирует закладку "Согласование" в форме "Настройка параметров подсистемы ТОИР" и в формах ввода документов для настройки и, непосредственно, согласования документов.
  • Использовать статусы документов ТОиР — включает использование статусов документов: "Выявленные дефекты" - зарегистрирован, запланировано устранение, устранен, отклонен"; "Внешнее основание для работ" - зарегистрирован, запланировано устранение, устраняется, устранен; "Смета ремонта (Заявка на ремонт)" - зарегистрирован, в работе, выполнен, отклонен; "Наряд на выполнение ремонтных работ" и "Наряд на регламентное мероприятие" - зарегистрирован, в работе, выполнен. Статусы документов используются в системе для оперативного отслеживания выполнения ремонтных работ в форме списка документов и в рабочем месте технического специалиста.
  • Использовать уведомления о событиях системы — включает всплывающие сообщения о событиях системы, в том числе по настраиваемому расписанию.
  • Использовать управление персоналом — включает доступ к кадровым документам "Прием на работу" и "Прием на работу списком", "Кадровый перевод" и "Кадровый перевод списком", "Увольнение" и "Увольнение списком".
  • Использовать настраиваемые представления — данный признак делает доступной закладку "Представление данных" в разделе "Настройки подсистемы ТОиР", в которой настраивается данный функционал при разрешении пользовательской настройки представлений.
    • Разрешить пользовательскую настройку представлений — функционал доступен при включении функции "Использовать настраиваемые представления" и перезапуска системы.
  • Показывать подсистему "Мониторинг показателей процессов" — данный признак делает доступной подсистему "Мониторинг показателей процессов" для регистрации значений внешних показателей, KPI и анализа показателей.
    • Автоматически обновлять поставляемую модель — автоматически загружать обновления поставляемой модели показателей при обновлении версии релиза 1С: ТОИР.

Подзакладка «Оборудование и нормативы»

  • Учет запчастей — включает возможность учета запчастей объектов ремонта в системе. В карточке объекта ремонта на закладке "Нормативы планирования" становится доступна подзакладка "Запчасти".
  • Использовать типовые объекты ремонта — данная настройка дает доступ к справочнику типовые объекты ремонта и делает возможной классификацию/типизацию оборудования. 
    • Использовать картинки типовых объектов — функционал доступен при включении функции "Использовать типовые объекты ремонта". Включает возможность использования изображений для типовых объектов. Выбранное изображение отобразится в иерархии объектов ремонта.
  • Использовать сложные технологические операции — при включении данного признака появляется возможность составления технологической карты не только из технологических операций, но и из иных технологических карт.

  • Использовать направления объектов ремонта — включает классификацию объектов ремонта по направлениям подразделений предприятия.
  • Использовать гарантии — включает использование механизма гарантии. В карточке объекта ремонта становится доступна соответствующая закладка для ввода информации по гарантии
    • Запретить ремонт гарантийного оборудования хозспособом — при включении данного признака система предложит создать Рекламационный акт при регистрации документа "Выявленные дефекты" и выведет напоминание о том, что оборудование находится на гарантии.
  • Использовать документы принятия и списания оборудования — при установке данного признака, в настройках параметров системы становится доступна закладка "Жизненный цикл оборудования".  В данной закладке настраивается доступность объекта ремонта в определенном статусе для подбора его в документы системы (смета, наряд, акт и др.).
  • Выделять инвестиционные/капитальные ремонты — делает доступным указание признака вида ремонта Инвестиционный/капитальный ремонт.

  • Использовать функциональные места — включает возможность создания типа объекта "Функциональное место". Функциональное место позволяет описать место установки оборудования внутри технологической цепочки производства на предприятии.

Подзакладка «Показатели эксплуатации»

  • Учет контролируемых показателей — включает возможность учета контролируемых показателей оборудования и соответствующий документ в подсистеме "Показатели эксплуатации".

    • Использовать ремонты по состоянию — включает функционал, позволяющий формировать выявленный дефект при выходе значения контролируемого показателя из допустимых границ.
    • Использовать стандартные оповещения о превышении контролируемых показателей — стандартные всплывающие окна с оповещением при вводе критических значений контролируемого показателя.
  • Учет наработки оборудования — включает возможность учета наработки оборудования и соответствующий документ в подсистеме "Показатели эксплуатации".
  • Учет состояния оборудования — включает возможность учета состояния оборудования и соответствующий документ в подсистеме "Показатели эксплуатации".
    • Учет пусков оборудования — включает функционал, который позволяет учитывать пуски и остановы оборудования.

    • Напоминать о необходимости зафиксировать простой при вводе документов — включает напоминание системы о необходимости зафиксировать простой объектов при вводе документов "Выявленные дефекты" и созданных на их основании документов "Акт о выполнении этапа работ".
  • Учет выявленных дефектов — данная опция включает возможность учета выявленных дефектов оборудования.
    • Использовать упрощенную форму дефекта — включение упрощенной формы ввода выявленного дефекта, в упрощенной форме отсутствует возможность добавления дефектов на подчиненные объекты ремонта.
    • Использовать анализ причин дефектов — включает инструмент, позволяющий проводить анализ причин возникновения дефектов. Становится доступен документ "Анализ причин дефекта" и обработка "Проведение анализа причин дефектов".
    • Назначение на ремонт с учетом рисков — включает возможность использования обработки "Матрица оценки рисков". Матрица оценки рисков позволяет обоснованно определить критичность дефекта и дату его устранения с применением риск-ориентированного подхода, который включает в себя определение возможного ущерба (людям, имуществу, окружающей среде, репутации, производству), тяжести последствий и вероятности возникновения последствий.
    • Использовать время учета рецидивности дефектов — определяет количество дней, в течение которых дефекты с одинаковыми параметрами считаются рецидивными. 

Подзакладка «Планирование ТОиР»

  • Использовать ППР — включает доступность документа "План-график ППР" в подсистеме "Планирование ТОиР".
    • Использовать фоновый расчет ППР — позволяет вести расчет плана-графика ППР в фоновом режиме и по настроенному расписанию.
    • Не пересчитывать ППР в фоне после ошибки — при включении этой опции, если в ходе расчета ППР в фоне произошла ошибка, повторные автоматические попытки расчета этого ППР в фоне по расписанию не будут выполняться, пока этот ППР заново не будет поставлен пользователем в очередь на фоновый расчет.
  • Учет замещаемых ремонтов — включает возможность замещения ремонтов при планировании. Активирует соответствующую закладку в карточке объекта ремонта в разделе "Нормативы планирования".

  • Использовать коэффициенты ремонтных особенностей — делает доступной подзакладку "Коэффициенты ремонтных особенностей" в карточке объекта ремонта и списке объектов регламентных мероприятий. На данной подзакладке отображаются данные по рассчитываемым и устанавливаемым коэффициентам ремонтных особенностей. Назначение коэффициентов объекту ремонта осуществляется путем создания и проведения документа "Установка коэффициентов ремонтных особенностей". Функционал использования коэффициентов, позволяет сокращать или увеличивать межремонтный интервал при выполнении построения план-графика ППР и графика регламентных мероприятий.

  • Использовать регламентные мероприятия — данная настройка включает доступность справочника "Списки объектов регламентных мероприятий" в подсистеме "Оборудование и нормативы" и документа "График регламентных мероприятий" в подсистеме "Планирование ТОиР".

  • Использовать остановочные ремонты  — включает возможность использования документа "Остановочные ремонты". Делает доступной подзакладку "Основное и зависимое оборудование" в карточке объекта ремонта. Документ позволяет сдвигать даты ремонтов зависимого и подчиненного по иерархии оборудования под заданный период ремонта основного оборудования для оптимизации проведения ремонтных работ на предприятии.
  • Использовать планирование по жестким ремонтным циклам — включение данной опции предполагает использование цепочек ремонтов для планирования по "жестко" заданному ремонтному циклу, указанному пользователем. Активируется закладка "Фиксированные ремонтные циклы" в нормативах планирования карточки объекта ремонта.

  • Использование лимитов ремонтов — позволяет использовать лимиты на плановые ремонтные работы, т.е. задавать определенную сумму затрат на ремонты, которую при планировании ППР нельзя превышать. Сумма определяется документом "Установка лимита на плановые ремонтные работы".

    • Использовать оценки риск-менеджеров — данный признак делает доступной обработку "Установка оценок риск-менеджмента", необходимую для установки оценок риск-менеджмента плановых ремонтов, т.е. для указания вероятности выхода из строя (экспертная вероятность выхода оборудования из строя в следующем ремонтном цикле, если не выполнить по плану ремонт этого вида), ущерба (сумма ущерба в национальной валюте, который будет нанесен, если оборудование выйдет из строя).

  • Рассчитывать плановую стоимость ремонтов при планировании — стоимость ремонта будет определяться на основании предполагаемых материальных затрат и трудовых затрат, указанных в технологических операциях планируемых ремонтов, на основании цен, введенных в документе "Установка цен номенклатуры" и стоимостей часа квалификаций, введенных в документе "Установка стоимостей квалификаций". 
  • Использовать лоты — активирует документ "Закупки (Лоты)", в котором выбирается подрядчик на основании Критериев выбора, определяемых документов "Оценка работы подрядчика", создаваемого на основании Акта выполнения этапа работ. В акте активируется кнопка "Заполнить из лотов" для подбора подрядчика, выполняющего ремонт. Также становится доступен документ "Оценка работы подрядчика". 
  • Использовать сметы ремонта — активирует документ "Смета ремонта. (Заявка на ремонт)". 
    • Использовать несколько смет по ремонту — позволяет вводить для одного ремонта несколько смет (заявок на ремонт). Если настройка отключена, система не даст ввести вторую смету.
  • Использовать общий план работ — делает доступной обработку "Общий план работ" в подсистеме "Планирование ТОиР".

    • Использовать скользящее планирование — позволяет настраивать виды скользящих планов (оперативный, среднесрочный, годовой и др.), выбирать вид плана в обработке "Общий план работ" для вывода информации в виде таблицы или диаграммы по видам работ и объектам ремонта для подразделения.

  • Использовать внешнее основание для работ — включает возможность использования Внешних оснований для проведения ремонтных работ.

  • Использовать приоритетность перенесенных ремонтов — активирует флаг "Перенесенный" в документе "План-график ППР. Корректировка". Флаг автоматически проставляется для ремонтов, даты которых были изменены в документе вручную или изменены с помощью обработки "Закрытие заявок и ремонтов". При пересчете корректировки даты ремонтов, для которых был установлен флаг "Перенесенный", не будут изменяться.

Подзакладка «Наряды и работы»

  • Использовать наряды на выполнение работ — включает возможность использования документа "Наряд на выполнение работ".
    • Наряд-допуск на работы повышенной опасности обязателен — для технологической операции указать признак "Работа повышенной опасности", после чего система будет требовать обязательного оформления документов "Наряд на выполнение ремонтных работ" для работ повышенной опасности.
  • Использовать опасное производство — появляется возможность для технологических операций указать признак "Опасная операция".

  • Использовать возможность отмены ремонтов — включает возможность отмены ремонтов через обработку "Закрытие заявок и ремонтов" или корректировку документа "План-график ППР". Может использоваться только вместе с настройками "Использовать ППР" и "Учет выявленных дефектов оборудования".

  • Использовать усложненную схему закрытия нарядов — включает возможность указывать в документах "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия" проценты выполнения ремонтных работ.

  • Запретить проведение актов с исполнителями без указания квалификаций — запрещает проведение акта о выполнении этапа работ без указания времени работы сотрудника по каждой технологической операции.

  • Запретить проведение актов без времени работы сотрудников — запрещает проведение документа "Акт о выполнении этапа работ" без указания времени работы сотрудника по каждой технологической операции.

  • Запретить пресечение времени работы сотрудников с одинаковой квалификацией — запрещает вводить пересекающееся время работы для сотрудников с одинаковой квалификацией по одной и той же технологической операции в нарядах и актах.
  • Использовать наряды на регламентное мероприятие — включает возможность использования документа "Наряд на регламентное мероприятие". Доступна, если включена настройка "Использовать регламентные мероприятия". 

  • Использовать акт приемки оборудования — включает возможность использования документа "Акт приемки оборудования".

  • Контролировать время работы в актах — при записи акта о выполнении этапа работ будет идти проверка на соответствие даты начала и окончания работ сотрудника временным рамкам самого ремонта.

  • Использовать комиссии — включает возможность указывать экспертные комиссии в документах. Состав указанной комиссии выводится на печатные формы документов.

Подзакладка «МТО»

  • Использовать контроль отрицательных остатков — включает возможность контролировать остатки номенклатуры при создании документов внутреннего потребления и перемещения товаров.
  • Использовать аналоги номенклатуры — позволяет использовать документ "Установка аналогов номенклатуры для ремонтов" и подбирать установленные им аналоги номенклатуры в документах "Смета ремонта (Заявка на ремонт)", "Акт о выполнении этапа работ".
  • Использовать характеристики номенклатуры — включает использование характеристик номенклатуры.
  • Использовать серии номенклатуры — включает использование серий номенклатуры.

  • Запретить ввод заказов на внутреннее потребление без заявки — запрещает вводить документ "Заказ на внутреннее потребление" без основания в виде "Сметы ремонта (заявки на ремонт)".
  • Запретить ввод внутреннего потребление без акта — запрещает вводить документ "Внутреннее потребление товаров" без основания в виде "Акта о выполнении этапа работ".

  • Ввод внутреннего потребления товаров только на основании проведенного акта — запрещает ввод документа "Внутреннее потребление товаров" на основании непроведенных документов "Акт о выполнении этапа работ" и "Акт о выполнении регламентного мероприятия".

  • Ввод оприходования товаров только на основании проведенного акта — запрещает ввод документа "Оприходование излишков товаров" на основании непроведенного документа "Акт о выполнении этапа работ". 
  • Ввод заказов на внутреннее потребление только на основании проведенного документа — выключает возможность создания документа "Заказ на внутреннее потребление" на основании непроведенных документов "План-график ППР", "График регламентных мероприятий", "Смета ремонта (Заявка на ремонт)".
  • Расширенный режим МТО — включает возможность автоматического создания заказов на внутреннее потребление для плановых ремонтов и поддержания их в актуальном состоянии.
    • Дата перехода на новую систему учета МТО — дата, с которой в системе начнется автоматическое создание заказов на внутреннее потребление.
  • Способ резервирования материалов — определяет порядок резервирования номенклатуры под заказы на внутреннее потребление в документе "Поступление товаров и услуг". "По ФИФО" ранжирует заказы по дате отгрузки, "С учетом критичности" исходит из приоритетности ремонта". 

Подзакладка «Интеграции»

  • Префикс информационной базы — префикс информационной базы ТОИР при синхронизации данных с другими программами.
  • Использовать синхронизацию данных — включает возможность настройки синхронизации данных с другими программами: 1С:ERP, 1С:УПП, 1С:RCM, 1С:ЗУП, 1С:УТ, мобильным приложением "Мобильная бригада" и АИС Диспетчер.
    • Интеграция заказов на ремонт в ERP — Включает возможность передавать в 1С: ERP заказы на ремонт, которые получаются из документов 1С:ТОИР "Выявленные дефекты", "План-график ППР" и "Внешнее основание для работ" и обновляются документами "Смета", "Наряд" и "Акт". 
    • Интеграция заказов на внутреннее потребление в ERP/УТ — включает возможность передавать в 1С: ERP / УТ заказы на внутреннее потребление. Доступна при включенной настройке "Использовать синхронизацию данных".
  • 1С: Производственная безопасность. Промышленная безопасность — включает возможность интеграции с "1С: Производственная безопасность. Промышленная безопасность".
  • Системы АСУ ТП — включает доступ к подсистеме "Интеграция с АСУ ТП".
  • Система мониторинга АИС "Диспетчер" — включает возможность использовать интеграцию с системой АИС "Диспетчер" для получения данных мониторинга из нее.
  • MS Project — включает использование команд выгрузки/загрузки списка ремонтов из документа "Смета ремонта (Заявка на ремонт)" и обработки "Общий план работ", списка зависимых ремонтов из документа "Остановочный ремонт" в MS Project (в виде задачи на ремонт).

Закладка «Согласования»

  • Использовать согласование документов ТОиР/регламентных мероприятий/МТО — признак, указывающий на использование согласований при проведении ремонтов/регламентных мероприятий/заказов поставщику.
  • Использовать согласование — признак,что документ будет участвовать в согласовании.

  • Вид документа согласования — наименование документа, который можно использовать в согласовании. 
  • Организация — наименование организации, в которой вводится согласование выбранного документа. 
  • Подразделение — наименование подразделения, в котором вводится согласование данного документа.
  • Способ согласования — наименование способа согласования. 
  • Запрет перевыбора — признак, запрещающий выбирать другой способ планирования в документе.

Закладка «Напоминания и уведомления»

  • Настройка системной учетной записи — открывает настройки учетной записи, которая будет использоваться системой для отправки и получения электронных писем.
  • Журнал уведомлений пользователей — открывает журнал уведомлений пользователей.
  • Параметры оповещения о проверке оборудования на функциональных местах — шаблон сообщения с параметрами оповещения, использующийся при проверке оборудования на функциональных местах.
  • Параметры оповещения о проверке требований функциональных мест по документам — шаблон сообщения с параметрами оповещения, использующийся при проверке требований функциональных мест по документам.
  • Вид документа — предопределённый список видов документов системы.
  • Событие — событие по выбранному виду документа.
  • Параметры оповещения — шаблон сообщения с параметрами оповещения, использующийся при наступлении события.

Использование кнопки [Настройка системной учетной записи электронной почты] открывает для настройки учетную запись, которая будет использоваться системой для отправки и получения электронных писем.

Закладка «Жизненный цикл оборудования»

  • Статус ОР в учете — название статуса объекта ремонта.
  • Доступность при подборе — признак, определяющий доступность статуса при подборе.
  • Флаг "Не участвует в планировании". Доступность флага — признак, определяющий будет ли для объекта ремонта отображаться флаг "Не участвует в планировании".
  • Флаг "Не участвует в планировании". Значение по умолчанию — признак, определяющий будет ли флаг "Не участвует в планировании" стоять по умолчанию.

    Закладка «Представление данных»

    • Объект конфигурации — наименование объекта конфигурации, которому будет меняться представление в системе.
    • Форматная строка —  форматная строка представления объекта конфигурации.
