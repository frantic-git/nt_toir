Рабочее место учета показателей эксплуатации

Рабочее место предназначено для быстрой и эффективной работы технического специалиста по учету показателей эксплуатации объектов ремонта.

В форме выводятся следующие поля для отбора:

  • Организация – отбор по организации-владельцу объекта ремонта.
  • Подразделение – отбор по подразделению-владельцу объекта ремонта.
  • Поле поиска – поле для указания параметра, по которому будет производиться поиск.
  • Содержит – поле для указания значения параметра поиска.
  • Направление – отбор по направлению объекта ремонта.
  • Типовой ОР – отбор по типовому объекту ремонта (ремонтной группе).
  • Критичность – отбор по критичности объекта ремонта.
  • Состояние – отбор по состоянию объекта ремонта.
  • Показатели – отбор по наполненности показателей у объекта ремонта.

Кнопка [Отчеты] позволяет перейти к формам отчетов: "Контролируемые показатели", "Наработка оборудования", "Простой оборудования", "Останов/запуск оборудования".

Кнопка [Документы] позволяет перейти к формам списка документов: "Учет контролируемых показателей", "Учет наработки оборудования", "Состояния объектов ремонта".

Кнопка [Настройка иерархии] позволяет выбрать структуру иерархии объектов ремонта в табличной части "Дерево".

Кнопка [Обновить] позволяет обновить данные рабочего места.

В табличной части "Дерево" выводятся следующие поля:

  • Наименование – наименование объекта ремонта или группы объектов ремонта.
  • Код – код объекта ремонта или группы.
  • Инвентарный № – инвентарный номер объекта ремонта.
  • Состояние – текущее состояние объекта ремонта.
  • К.П. – признак наличия данных по контролируемым показателям у объекта ремонта.
  • П.Н. – признак наличия данных по показателям наработки у объекта ремонта.

Группа «Данные текущего объекта ремонта»

Предназначена для вывода данных об объекте, выбранном в табличной части "Дерево".
  • Наименование – наименование объекта ремонта.
  • Направление – наименование направления объекта ремонта.
  • Критичность – наименование критичности объекта ремонта.
  • Инвентарный № – инвентарный номер объекта ремонта.
  • Технологический № – технологический номер изделия–объекта ремонта.
  • Типовой ОР – наименование типового объекта ремонта (ремонтной группы).
  • Состояние ОР – состояние объекта ремонта (зависит от документа "Состояния объектов ремонта").
  • Подразделение – наименование подразделения-владельца объекта ремонта.
  • Заводской № – номер изделия–объекта ремонта, установленный на заводе по номенклатуре объектов.
  • Причина простоя – причина простоя (заполняется, если состояние ОР – простой).

    Кнопка [Создать документы] позволяет автоматически создавать документы "Учет наработки оборудования", "Учет состояний оборудования" и "Учет контролируемых показателей" при наличии включенных соответствующих функциональных опций.

    Группа «Параметры»

    Предназначена для настройки учета иерархии элементов.

    Группа «Наработка»

    Содержит настройки, отвечающие за формирование табличной части "Учет фактической наработки". 

    • Запрашивать список показателей наработки – при добавлении фактической наработки будет предварительно отображаться список показателей наработки объекта ремонта.
    • Групповое формирование наработки – при добавлении фактической наработки для нескольких объектов ремонта позволяет ввести наработку одним документом.
    • Проставлять плановую наработку – позволяет считать фактическую наработку по плановой, ее значение не нужно вводить вручную.
    • Учитывать простои ОР – позволяет не учитывать фактическую наработку в периоды простоя объекта ремонта.

    Группа «Контролируемые показатели»

    Содержит настройки, отвечающие за формирование табличной части "Учет контролируемых показателей". 

    • Запрашивать список контролируемых показателей – при добавлении контролируемого показателя будет предварительно отображаться список контролируемых показателей объекта ремонта.
    • Групповое формирование контролируемых показателей – при добавлении контролируемых показателей для нескольких объектов ремонта позволяет ввести одним документом.
    • Дата – дата, на которую формируется документ "Учет контролируемых показателей".

      Группа «Состояния»

      Содержит настройки, отвечающие за формирование табличной части "Учет фактического состояния". 

      • Групповое формирование состояний – при добавлении фактического состояния для нескольких объектов ремонта позволяет ввести фактическое состояние одним документом.
      • Дата – дата, на которую формируется документ "Состояние объектов ремонта".

        Группа «Учет фактической наработки»

        Табличная часть "Учет наработки" предназначена для ввода фактической наработки.

        • Объект ремонта – наименование объекта ремонта, для которого регистрируется наработка.
        • Показатель – обозначение показателя, в разрезе которого фиксируется наработка объекта ремонта.
        • Ед. изм. – обозначение показателя, в разрезе которого фиксируется наработка объекта ремонта.
        • Дата работы с – значение определяет начало периода, за который регистрируется наработка объекта ремонта.
        • Текущее значение – значение наработки на текущую дату (если ранее наработка не фиксировалась по данному показателю, то поле отображается незаполненным).
        • Наработка – значение параметра наработки, которое будет зарегистрировано для объекта ремонта.
        • Дата работы по – значение определяет окончание периода, за который регистрируется наработка объекта ремонта.
        • Новое значение – новое значение наработки объекта ремонта.
        • Распространять на подчиненные ОР – признак, определяющий возможность назначить подчиненному объекту ремонта ту же наработку, что и основному. При установленном признаке наработка, введённая объекту ремонта, автоматически присвоится и всем подчинённым объектам ремонта.
        • Синхронизировать – если признак установлен, всем подчинённым объектам ремонта можно автоматически присвоить значение наработки основного объекта ремонта (с помощью кнопки Синхронизировать).

          Кнопка [Синхронизировать] позволяет автоматически присвоить значение наработки основного объекта ремонта подчиненному.

          Группа «Учет фактического состояния»

          Табличная часть "Учет состояний" предназначена для ввода состояний объекта ремонта.
          • Объект ремонта – наименование объекта ремонта, для которого регистрируется изменение состояния его эксплуатации.
          • Распространять на подчиненные – признак того, что состояние объектов ремонта, входящих в состав вводимого объекта ремонта, будет также изменено.
          • Дата изменения состояния – дата, на которую необходимо ввести изменение состояние оборудования.
          • Вид эксплуатации – вид эксплуатации объекта ремонта с даты изменения состояния.
          • Причина простоя – заполняется для вида эксплуатации "простой".

          Кнопка [Ввод состояния] позволяет ввести состояние на дату (если известна только дата начала состояния) или на период (если известен весь период состояния).

          Группа «Учет контролируемых показателей»

          Табличная часть "Учет контролируемых показателей" предназначена для ввода контролируемых показателей.

          • Объект ремонта – название объекта ремонта, для которого регистрируется значения контролируемых показателей.
          • Дата контроля – дата ввода значений контролируемого показателя (заполняется автоматически значение из группы "Параметры – Контролируемые показатели").
          • Показатель – наименование контролируемого показателя.
          • Ед. изм. – единица измерения контролируемого показателя.
          • Значение – значение контролируемого показателя.

          Кнопка [Добавить показатель] позволяет добавить в табличную часть только объект ремонта, без автоматического заполнения контролируемых показателей.
