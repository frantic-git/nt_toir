Обработка «Проведение анализа причин дефектов»

Обработка предназначена для определения дефектов, по которым еще не выполнялся анализ причин дефектов и по которым уже начат. Для этого предназначена закладка "Анализ не начат", в ней находится информация по документам "Выявленные дефекты". На закладке "Анализ начат" выводится список начатых анализов на основе данных документов "Выявленные дефекты" и "Анализ причин дефекта".

В форме выводятся следующие поля:

  • Период - отбор по дате регистрации выявленного дефекта.
  • Организация - отбор по организации, в которой зарегистрирован дефект. По умолчанию заполняется из пользовательских настроек.
  • Подразделение - отбор по подразделению, в котором зарегистрирован дефект. По умолчанию заполняется из пользовательских настроек.
  • Объект ремонта - отбор по объекту ремонта, в котором обнаружен дефект.
  • Статус дефекта - отбор по статусу дефекта. Может принимать 4 значения: зарегистрирован, запланировано устранение, устраняется, устранен.
  • Критичность - отбор по критичности дефекта.
  • Статус анализа - отбор по статусам анализа причин дефекта. Может принимать 4 значения: зарегистрирован, в работе, анализ выполнен, закрыт.

Закладка «Анализ не начат»

  • Дата - дата регистрации выявленного дефекта.
  • Выявленный дефект - ссылка на документ "Выявленные дефекты".
  • Объект ремонта - наименование объекта ремонта.
  • Описание дефекта- краткое описание дефекта.
  • Отказавший элемент - наименование ремонтной единицы, для которой зарегистрирована неисправность.
  • Статус дефекта - текущий статус документа "Выявленные дефекты".

Кнопка [Начать анализ] предназначена для перехода к форме документа "Анализ причин дефекта".

Закладка «Анализ начат»

На форме содержатся 2 табличные части - Список начатых анализов и Список корректирующих мероприятий для выбранного анализа.

В табличной части Список начатых анализов выводятся следующие поля:

  • Дата - дата регистрации анализа.
  • Анализ - ссылка на документ "Анализ причин дефекта".
  • Выявленный дефект - ссылка на документ "Выявленные дефекты".
  • Критичность - критичность дефекта (поле "Критичность дефекта" из документа "Выявленные дефекты").
  • Мероприятия завершены - флаг, устанавливающийся в истину, если для данного анализа было назначено хотя бы одно корректирующее мероприятие, и все они находятся в конечном статусе (выполнены или отменены).
  • Непосредственная причина - наименование непосредственной причины возникновения дефекта (поле "Непосредственная причина дефекта" из документа "Выявленные дефекты").
  • Объект ремонта - наименование объекта ремонта.
  • Описание - краткое описание дефекта (поле "Описание дефекта" из документа "Выявленные дефекты").
  • Отказавший элемент - наименование ремонтной единицы, для которой зарегистрирована неисправность.
  • Статус анализа - статус документа "Анализ причин дефекта".
  • Статус дефекта - текущий статус документа "Выявленные дефекты".

В табличной части Список корректирующих мероприятий выводятся следующие поля:

  • Мероприятия завершены - флаг, устанавливающийся в истину, если по данному корректирующему мероприятию завершены все действия (находится в конечном статусе).
  • Описание - дополнительная текстовая информация, заполняемая из описания корректирующего мероприятия.
  • Ответственный - ФИО сотрудника, ответственного за выполнение мероприятия.
  • Срок - плановый срок выполнения работ.
  • Внешнее основание/задача - ссылка на документ "Внешнее основание" или бизнес-процесс "Задание".
  • Статус - статус ремонта (если по документу "Внешнее основание для работ" введен документ "Смета ремонта (Заявка на ремонт)"), либо документа "Внешнее основание для работ" (при отсутствии "Сметы ремонта (Заявки на ремонт)"), либо состояние бизнес-процесса.
