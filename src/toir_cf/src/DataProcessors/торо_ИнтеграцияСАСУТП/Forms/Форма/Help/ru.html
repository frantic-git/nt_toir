Обработка «Интеграция с АСУ ТП»

Закладка «Настройки»

  • Удалять обработанные записи - при установке этого флага, обработанные записи будут удалены.
  • Отступ - вычитается из даты начала сеанса, которая передается в запросы пометки, чтения и удаления.
  • Оповещаемые лица - список пользователей, в очередь уведомлений которых добавляется информация об ошибках интеграции, произошедших в рамках текущего сеанса. Впоследствии, информация отправляется им посредством системной учётной записи электронной почты.

 

Закладка «Отладка»

  • Загрузка данных из макета - при установке этого флага, данные загружаются не из базы данных, а из специального макета «торо_ОтладочныеДанные», в котором содержится область «Данные».
  • Получить данные - инициируется сеанс получения связи (вручную или посредством регламентного задания).
  • Обработать данные - инициируется сеанс обработки данных (вручную или посредством регламентного задания, используется то же регламентное задание, что и для получения данных).

 

Закладка «Загрузки»

Для повышения удобства начальной настройки интеграции добавлена возможность загрузки соответствий в регистр «Соответствия тегов объектам ТОИР»

  • Имя файла - файл, из которого будет происходить загрузка. Файл должен содержать следующие колонки:
    1. Наименование ОР – произвольная строка для установления соответствия;
    2. Тег - произвольная строка для установления соответствия;
    3. Подключение – наименование элемента из справочника «Подключение»;
    4. Объект ремонта – наименование элемента из справочника «Объекты ремонта»;
    5. Тип сигнала – наименование элемента из перечисления «типы сигналов АСУ ТП»;
    6. Показатель – наименование элемента из плана видов характеристик «Измеряемые показатели объектов ремонта»;
    7. Параметр наработки - наименование элемента из справочника «Параметры выработки ОР»;
    8. Состояние – наименование элемента из справочника «Виды эксплуатации»;
    9. Отключить – текстовая строка, которая должна содержать строку «Истина» или «Ложь»;
    10. Ответственный – наименование элемента из справочника «Пользователи»;
    11. Подразделение – наименование элемента из справочника «Структура предприятия»;
    12. Организация – наименование элемента из справочника «Организации»;
    13. Причина простоя – наименование элемента из справочника «Причины простоя оборудования»;
    14. Типовой дефект – наименование элемента из справочника «Типовые дефекты оборудования»;
    15. Показатель KPI – наименование элемента из справочника «Показатели KPI».
  • При загрузке происходит контроль корректности загружаемых записей, если содержимое колонок 4-8 и 10-15 не сможет сопоставиться с объектами системы - запись будет считаться ошибочной и не будет загружена в систему. Кроме того происходит контроль дублей (контроль уникальности записей определятся по колонкам 1-3), записи дублирующие уже имеющиеся не будут загружены в систему.
