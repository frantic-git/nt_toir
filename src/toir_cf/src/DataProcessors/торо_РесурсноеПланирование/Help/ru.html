Обработка «Ресурсное планирование работ»

Обработка используется сотрудником, отвечающим за распределение нагрузки на сотрудников, для формирования документов "Наряд на выполнение ремонтных работ" и назначения исполнителей работ.

 

В форме используются следующие отборы:

  • Организация — наименование организации, отвечающей за ремонт.
  • Направление — наименование направления объекта ремонта.
  • Исполнитель — наименование подразделения, выполняющего ремонт.

С помощью кнопки [Обновить] обновляются данные и устанавливаются отборы. При нажатии кнопки [Настройки], откроется дополнительная форма, в которой возможно более тонко настроить отображаемые ремонты.

Раздел «Ремонты оборудования»

 

В таблице используется различное оформление для разных строк.

Измененные, но не сохраненные строки выводятся голубым шрифтом (а над таблицей появляется надпись "Есть несохраненные изменения").

Обеспеченные ремонты выводятся зеленым.

Завершенные ремонты — серым.


При необходимости возможно убрать неиспользуемые колонки таблицы с помощью команды [Изменить форму...], находящуюся в подменю [Еще].

  • Есть несохраненные изменения - флаг, указывающий на присутствие изменений по текущей строке ремонта.
  • Ремонт завершен - отметка о завершенном ремонте.
  • Ремонт обеспечен исполнителями — отметка об обеспечении ремонта исполнителями.
  • Гарантийный ремонт - колонка отображается только, если в системе используются гарантийные ремонты. Флаг, указывающий, что на дату ремонта есть действующая гарантия.
  • Дата начала — дата начала ремонта.
  • Дата окончания — дата окончания ремонта.
  • Объект ремонта — ссылка на объект ремонта.
  • Вид ремонта — вид ремонта.
  • Документ начала цепочки — ссылка на документ начала цепочки ремонта.
  • Статус ремонта — текущий статус ремонта.
  • Исполнитель — подразделение, выполняющего ремонт.
  • Подразделение — подразделение-владелец объекта ремонта.
  • Организация — организация объекта ремонта.
  • Описание дефекта — текстовое описание дефекта.
  • Критичность дефекта — критичность дефекта.
  • Состояние ОР — текущее состояние объекта ремонта.
  • Направление — направление объекта ремонта.
  • Критичность — критичность объекта ремонта.

Раздел «Дерево ремонтных работ»

 

В таблице используется различное оформление для разных строк.

Измененные, но не сохраненные строки выводятся голубым шрифтом (а над таблицей Ремонтов оборудования появляется надпись "Есть несохраненные изменения").

Обеспеченные ремонтные работы выводятся зеленым.

Завершенные ремонтные работы — серым.

 

С помощью кнопки [Добавить технологическую карту ремонта] можно добавить произвольную технологическую карту.


С помощью кнопки [Добавить технологическую операцию] можно добавить произвольную технологическую операцию.


С помощью кнопки [Указать параллельно выполняемые] можно указать параллельно выполняемые ремонтные работы.


  • Ремонтная работа обеспечена исполнителями — отметка об обеспечении ремонтной работы исполнителями.
  • Есть акт - флаг, указывающий на введенный акт по ремонтной работе. Двойной клик по флагу открывает акт.
  • Ремонтная работа — наименование ремонтной работы.
  • Количество — количество исполняемых ремонтных работ.
  • Дата начала — рассчитанная прогнозируемая дата начала ремонтной работы. Рассчитывается по круглосуточному графику с учетом параллельно выполняемых операций.
  • Норма — норма времени выполнения ремонтной работы.
  • Картинка "Стрелка влево" — указывает по какой ремонтной работе выделенный исполнитель.

Раздел «Исполнители работ»

 

С помощью кнопки [Добавить] можно добавить новую строку с исполнителем ремонтной работы.


Кнопки [Разбить работу] выполняет разбитие работы на интервалы согласно рабочему времени по графику работы сотрудника.


  • Квалификация — наименование квалификации, требующейся для выполнения ремонтной работы.
  • Сотрудник — ФИО сотрудника, который назначен на выполнение этой ремонтной работы. Заполняется путем переноса сотрудника мышкой из соседнего списка "Ресурс исполнителей".
  • Дата начала — дата начала выполнения ремонтной работы.
  • Дата окончания — дата окончания выполнения ремонтной работы.
  • Время работы — время, требующееся для выполнения ремонтной работы. Если оно больше длины рабочего дня — подсвечивается желтым цветом.
  • Интервал работ выходит за пределы дня — отметка о том, что длительность ремонта превышает длительность рабочего дня и требуется разбить строки.

Раздел «Ресурс исполнителей»

 

При подборе исполнителей черным цветом отображаются сотрудники, имеющие доступное для выполнения работ время и необходимую для выбранной работы квалификацию. Серым цветом отображаются сотрудники, ресурс которых уже занят или они не имеют необходимую квалификацию. Но и таких сотрудников тоже можно назначить исполнителями на выполнение работ.

  

С помощью кнопки [График работы сотрудника] можно просмотреть график работы сотрудника.

 

С помощью кнопки [Список назначенных работ сотрудника] можно посмотреть список запланированных работ сотрудника.


  • Сотрудник — ФИО сотрудника организации.
  • Квалификация — квалификация сотрудника.
  • Основная квалификация — отметка о том, что указанная квалификация является основной.
  • Назначено работ, ч — суммарное количество часов назначенных работ на выбранный день ремонта.
  • Плановая доступность, ч — время в часах плановой доступности сотрудника в выбранный день ремонта.
  • Доступное время, ч — доступное для распределения время в часах.

 

Надпись Документ-источник отображает источник данных для исполнителей и ремонтных работ.

 

С помощью кнопки [Применить изменения] можно сохранить изменения в документ "Наряд на выполнение ремонтных работы".

С помощью кнопки [Перейти к документам] можно перейти к списку созданных в текущем рабочем сеансе документов.
