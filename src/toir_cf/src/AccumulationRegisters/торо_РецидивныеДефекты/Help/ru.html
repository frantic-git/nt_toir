Регистр накопления "Рецидивные дефекты"

Регистр предназначен для хранения информации о рецидивных дефектах в разрезе объектов ремонта.

Рецидивный дефект определяется точным совпадением параметров с одним из предыдущих дефектов (у которых дата обнаружения меньше, чем у текущего):

  • Объект ремонта + Отказавший элемент + Типовой дефект (если типовой дефект заполнен);
  • Объект ремонта + Отказавший элемент + Вид дефекта + Причина дефекта (если типовой дефект не заполнен).

Движения в регистре происходя при проведение документа "Выявленные дефекты", если в нем перечислены фактические затраты номенклатуры.

Измерения:

  • Объект ремонта - отказавший элемент из документа выявленных дефектов.
Ресурсы:
  • Количество - количество рецидивных дефектов оборудования. При формировании движений всегда равно 1. При расчете показателя KPI "Количество рецидивных дефектов" извлекаются обороты по этому регистру и получается корректное количество рецидивных дефектов по определенному объекту ремонту.
Реквизиты:
  • ID - ID рецидивного дефекта;
  • IDПредыдущегоДефекта - ID предыдущего дефекта (на основании которого текущий считается рецидивным).

Данные регистра используются при формирование отчета "Рецидивные дефекты" и при расчете показателя KPI "Количество рецидивных дефектов".
