Отчет «План-фактный анализ затрат МТО»

Отчет предназначен для план-фактного анализа трудозатрат по ремонтным работам. Отчет содержит перечень объектов ремонта предприятия, по которым выполнялись ремонты за указанный период. Для каждого объекта ремонта выводится вид ремонта и его плановая дата начала. Затраты, задействованные при выполнении ремонта группируются на материальные и запчасти.

Открывается по команде План-фактный анализ затрат МТО:

  • в списке Отчеты по МТО подсистемы МТО;
  • в списке Отчеты по нарядам и работам подсистемы Наряды и работы. 

Параметры отчета 

Обращение к режиму настройки отчета выполняется с помощью кнопки Настройки. В отчете можно настроить отборы путем установки значений параметров отчета:

  • Период отчета — отбор материальных затрат по плановым датам;
  • Структура иерархии — отбор, позволяющий вывести полный путь к объекту ремонта в зависимости от выбранной иерархии. По умолчанию заполняется из пользовательских настроек (настройка "Основная структура иерархии");
  • Вид документа — настройка, позволяющая отображать материальные затраты по виду документа.
  • Учитывать сметы (заявки на ремонт) по дефектам — настройка, позволяющая учитывать затраты по документам "Выявленные дефекты";
  • Учитывать план-графики ППР — настройка, позволяющая учитывать затраты по документам "План-график ППР";
  • Учитывать сметы (заявки на ремонт) по внешним основаниям — настройка, позволяющая учитывать затраты по документам "Внешнее основания для работ";
  • Учитывать регламентные мероприятия — настройка, позволяющая учитывать затраты по документам  "График регламентных мероприятий".

Варианты отчета 

С помощью кнопки Варианты отчета можно открыть список доступных вариантов отчета. Доступны два предопределенных варианта отчета: 

  1. План-фактный анализ затрат МТО — открывается по умолчанию. Данные о материальных затратах сгруппированы по объектам ремонта. Фиксация плановых значений осуществляется документами "Смета ремонта (Заявка на ремонт)", "План-график ППР", "График регламентных мероприятий", фактических - документами  "Акт о выполнении этапа работ", "Акт о выполнении регламентного мероприятия":
    • Объект ремонта — наименование объекта ремонта; 
    • Вид ремонта — вид ремонта (поле "Вид ремонта" из документа "План-график ППР"; поле "Вид мероприятия" из документа "График регламентных мероприятий"; поля "Вид ремонта при вводе на основании выявленных дефектов" и "Вид ремонта при вводе на основании внешних оснований" из настроек параметров подсистемы ТОИР);

    • Дата начала план — планируемая дата начала ремонтных работ (поле "Дата начала" из документа "Смета ремонта (Заявка на ремонт)" для документов "Внешнее основание для работ", "Выявленные дефекты"; поле "Период" из документов "План-график ППР", "График регламентных мероприятий" для документов "План-график ППР" и "График регламентных мероприятий");
    • Вид затрат — общие затраты, разделенные на две группы: материальные, запчасти;
    • Номенклатура — наименование номенклатуры, которая используется для выполнения ремонтной работы (поле "Номенклатура" из документа "Смета ремонта (Заявка на ремонт)" на закладке "Материальные затраты" и "Запчасти" для документов "Внешнее основание для работ", "Выявленные дефекты"; поле "Номенклатура" согласно полю "Технологическая карта" из справочника "Объекты ремонта" на вкладке "Нормативы планирования - Нормативные ремонты/Запчасти" для документов "План-график ППР", "График регламентных мероприятий"):
      • Характеристика номенклатуры — наименование характеристики номенклатуры (поле "Характеристика" из документа "Смета ремонта (Заявка на ремонт)" для документов "Внешнее основание для работ", "Выявленные дефекты"; поле "Характеристика" согласно полю "Технологическая карта" из справочника "Объекты ремонта" для документов "План-график ППР", "График регламентных мероприятий"). Выводится, если установлена настройка системы в разделе Настройка параметров подсистемы ТОиР - Интерфейс и ввод документов - МТО: Использовать характеристики номенклатуры;
      • Единица измерения— наименование единицы измерения номенклатуры, требуемой для выполнения ремонтной работы (поле "Ед. изм." из документов "Акт о выполнении этапа работ", "Акт о выполнении регламентного мероприятия" на закладке "Материальные затраты");
    • Плановое количество — количество, которое запланировано для выполнения ремонтной работы (поле "Количество" из документа "Смета ремонта (Заявка на ремонт)" на закладках "Материальные затраты", "Запчасти" для документов "Внешнее основание для работ", "Выявленные дефекты"; поле "Количество" согласно полю "Технологическая карта" из справочника "Объекты ремонта" для документов "План-график ППР", "График регламентных мероприятий");
    • Плановая сумма — сумма плановых затрат на материальные ресурсы, которая потребуется для выполнения ремонтной работы (рассчитывается по формуле: Плановое количество * Цена на плановую дату ремонта. Цена номенклатуры устанавливается документом "Установка цен номенклатуры");
    • Фактическое количество — количество, которое фактически затрачено на выполнение ремонтной работы (поле "Количество" из документов "Акт о выполнении этапа работ", "Акт о выполнении регламентного мероприятия" на закладках "Материальные затраты", "Запчасти");
    • Фактическая сумма — сумма фактических затрат на материальные ресурсы, которая потребовалась для выполнения ремонтной работы (рассчитывается по формуле: Фактическое количество * Цена на фактическую дату ремонта);
    • Отклонение количество — рассчитывается как разность между полями "Плановое количество" и "Фактическое количество";
    • Отклонение сумма — рассчитывается как разность между полями "Плановая сумма" и "Фактическая сумма". 
  2. План-фактный анализ затрат МТО (с иерархией) — вариант отчета, в котором материальные затраты ремонтов сгруппированы по объекту иерархии, объекту ремонта:
    • Объект иерархии — наименования всех родителей объекта ремонта (объекты ремонта или группы объектов ремонта).
Для изменения структуры данных отчета или настройки отборов необходимо выполнить обращение к режиму настройки отчета с помощью кнопки Настройки.
